interface

area<%> : interface?

area<%>对象是用于管理其他区域位置和大小的窗口或无窗口容器。area<%>可以是容器、窗格或两者。唯一没有父级的区域是顶级窗口。

所有area<%>类都接受以下命名的实例化参数：

  • min-width——默认为初始图形最小宽度；传递到最小宽度min-width

  • min-height——默认为初始图形最小高度；传递到最小高度min-height

  • stretchable-width——默认值特定于类；传递到可拉伸宽度stretchable-width

  • stretchable-height——默认值特定于类；传递到可拉伸高度stretchable-height

method

(send an-area get-graphical-min-size)

 → 
dimension-integer? dimension-integer?

以两个值返回区域的图形最小大小：最小宽度和最小高度(以像素为单位)。

有关更多信息，请参见几何管理。请注意，返回值不取决于区域的min-width和min-height设置。

method

(send an-area get-parent)

 → (or/c (is-a?/c area-container<%>) #f)

返回区域的父级。顶级窗口可能没有父级(在这种情况下返回#f),或者它可能有另一个顶级窗口作为其父级。

method

(send an-area get-top-level-window)

 → (or/c (is-a?/c frame%) (is-a?/c dialog%))

返回区域最近的框架或对话框原型。对于框架或对话框区域，将返回框架或对话框本身。

method

(send an-area min-width) → dimension-integer?

(send an-area min-width w) → void?
  w : dimension-integer?

获取或设置用于几何图形管理的区域最小宽度(像素)。

如果最小宽度小于区域的图形最小宽度，或者如果最小宽度小于container-size报告的宽度(如果区域是容器)，则忽略最小宽度。有关更多信息，请参见几何管理。

区域的初始最小宽度是其图形最小宽度。另请参见get-graphical-min-size。

设置最小宽度时，如果w小于内部强制最小值，an exn:fail:contract exception is raised。

method

(send an-area min-height) → dimension-integer?

(send an-area min-height h) → void?
  h : dimension-integer?

获取或设置用于几何图形管理的区域的最小高度。

当最小高度小于区域的图形最小高度时，或当最小高度小于container-size报告的高度(如果区域是容器)时，忽略最小高度。有关更多信息，请参见几何管理。

区域的初始最小高度是图形的最小高度。另请参见get-graphical-min-size。

设置最小高度时(以像素为单位)；如果h小于内部强制最小值，an exn:fail:contract exception is raised。

method

(send an-area stretchable-height) → boolean?

(send an-area stretchable-height stretch?) → void?
  stretch? : any/c

获取或设置用于几何图形管理的区域的垂直可伸缩性。有关更多信息，请参见几何管理。

method

(send an-area stretchable-width) → boolean?

(send an-area stretchable-width stretch?) → void?
  stretch? : any/c

获取或设置用于几何图形管理的区域的水平可伸缩性。有关更多信息，请参见几何管理。
