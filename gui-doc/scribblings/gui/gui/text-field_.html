class

text-field% : class?

  superclass: object%

  extends: control<%>

text-field%对象是一个可编辑的文本字段，其前面显示可选标签。有两种文本字段样式：

  • 一行文本可见，当用户按回车键或回车键（文本字段具有焦点）且文本字段的框架或对话框未处理该事件时，将生成一个特殊的控件事件（请参见on-traverse-char in top-level-window<%>）。

  • 多行文本可见，不专门处理输入。

每当用户更改文本字段的内容时，都会调用其回调过程。在创建每个文本字段时，回调过程作为初始化参数提供。

文本字段使用text%编辑器实现（具有不可访问的显示）。因此，虽然text-field%只提供get-value和set-value来操作文本字段中的文本，但get-editor返回字段的编辑器，该编辑器为对文本进行更复杂的操作提供了大量的方法集合。

通过调用由current-text-keymap-initializer参数确定的当前键映射初始值设定项过程来初始化文本字段编辑器的键映射。

如果label不是#f，则用作文本字段标签。否则，文本字段不显示其标签。

如果&出现在label中，则特别解析为button%。

当用户更改文本字段中的文本或按Enter键时，调用callback过程（Enter不由文本字段的框架或对话框处理；请参见on-traverse-char in top-level-window<%>）。如果用户按Enter键，则传递给回调的事件类型为'text-field-enter，否则为'text-field。

如果init-value不是""，文本项的图形最小大小（graphical minimum size）将足够宽以显示init-value。否则，将选择内置默认宽度。对于单行模式下的文本字段，图形最小大小设置为显示一行，默认情况下，只有控件的宽度可以拉伸。对于多行文本字段，图形最小大小显示三行文本，默认情况下，它可以在两个方向上拉伸。

样式必须正好包含'single或'multiple中的一个；前者指定单行字段，后者指定多行字段。'hscroll样式仅适用于多行字段；当指定'hscroll时，该字段具有水平滚动条，并且禁用自动换行；否则，该字段没有水平滚动条，并且启用自动换行。多行文本字段始终具有垂直滚动条。'password指示字段应使用通用符号而不是实际字符绘制其内容的每个字符。如果style包括'vertical-label，则文本字段使用控件上方的标签创建；如果style不包括'vertical-label（可选包括'horizontal-label），然后标签创建在文本字段的左侧。 如果style包括'deleted，则文本字段将创建为隐藏的，并且它不会影响其父级的几何体；文本字段稍后可以通过调用parent的add-child方法激活。

font参数确定控件的字体。 有关enabled参数的信息，请参见window<%>。 有关horiz-margin和vert-margin的信息参数，请参见subarea<%>。 有关min-width、min-height、stretchable-width、stretchable-height以及stretchable-height参数，请参见area<%>。

method

(send a-text-field get-editor) → (is-a?/c text%)

返回用于实现文本字段的编辑器。

对于文本字段，text%对象最有用的方法如下：

  • (send a-text get-text)返回编辑器的当前文本。

  • (send a-text erase)从编辑器中删除所有文本。

  • (send a-text insert str)在当前插入符号位置将str插入编辑器。

method

(send a-text-field set-value val) → void?

  val : string?

设置文本字段中当前的文本。（未调用控件的回调过程。）

文本字段值可以被用户键入控件更改，这样的更改不会通过这个方法；使用control回调过程（作为初始化参数提供）来监视值更改。
