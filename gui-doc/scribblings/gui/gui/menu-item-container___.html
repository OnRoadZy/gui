interface

menu-item-container<%> : interface?

menu-item-container<%>对象是menu%、popup-menu%或menu-bar%。

method

(send a-menu-item-container get-items)

 → (listof (is-a?/c menu-item<%>))
返回菜单、弹出菜单或菜单栏中的项目列表。返回列表中项目的顺序与用户在菜单或菜单栏中看到的顺序相对应。

method

(send a-menu-item-container on-demand) → void?

Specification:

规范：当用户单击容器作为菜单栏时调用（在用户看到任何菜单项之前，除了下面指出的Unity的全局菜单栏），在容器作为弹出菜单弹出之前，或在检查包含该项的菜单栏以进行快捷键绑定之前。

如果容器不是菜单栏或弹出菜单，则通常通过容器所属菜单栏或弹出菜单的on-demand方法调用此方法，因为方法的默认实现链接到其项的on-demand方法。但是，该方法可以在容器中重写，这样它就不会调用其项的on-demand方法。

在使用统一窗口管理器的Unix上，使用全局菜单栏（Ubuntu上的默认菜单栏），当用户单击菜单栏时，racket/gui/base不会收到任何通知。为了估计用户点击菜单栏触发的on-demand，每当菜单栏的frame%对象失去键盘焦点时，就会调用on-demand。请注意，如果键盘焦点由于单击菜单而丢失，那么在on-demand调用期间添加到单击菜单的项可能不会为用户显示。

Default implementation:

默认实现：调用创建对象时提供的demand-callback过程，然后调用所包含项的on-demand方法。
