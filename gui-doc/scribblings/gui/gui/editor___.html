method

(send an-editor add-canvas canvas) → void?

  canvas : (is-a?/c editor-canvas%)

将画布添加到此编辑器的显示画布列表中。（参见get-canvases。）

通常，此方法只能由set-editor in editor-canvas%调用设置编辑器。

method

(send an-editor add-undo undoer) → void?

  undoer : (-> any)

将撤消过程添加到编辑器的撤消堆栈中。如果当前正在执行撤消操作，则会将撤消操作添加到编辑器的重做堆栈中。当撤消（或重做）更改为编辑器时，以及当此撤消是撤消（或重做）堆栈上的第一个项目时，系统将调用撤消程序。

编辑器实例是在没有撤消堆栈的情况下创建的，因此除非使用set-max-undo-history更改撤消堆栈的大小，否则不会记录撤消操作。

系统自动安装撤消记录以撤消内置的编辑器操作，如插入、删除和字体更改。只有在需要维护非内置状态或处理非内置操作时，才安装撤销器。例如，在用户可以为粘贴板中的剪切分配标签的程序中，该程序应该安装一个撤销器以恢复标签更改。因此，当用户更改剪切的标签，然后（从标准菜单栏）选择“撤消”时，剪切的标签将按预期恢复。相反，当用户通过拖动剪切来移动剪切时，不需要安装撤消器，因为系统会自动安装适当的撤消器。

撤消器返回后，将从编辑器的撤消（或重做）堆栈中弹出撤消器；如果返回值为真，则下一个撤消器也将作为同一撤消（或重做）步骤的一部分执行。如果要撤消的操作最初是作为begin-edit-sequence和end-edit-sequence的一部分执行的，则撤消器应返回真。如果撤消的操作是序列的隐式部分，则返回值也应为真。要扩展上一个示例，如果标签更改与移动配对以重新对齐剪切，则应在调用move后将标签更改撤消对象添加到编辑器中，并在调用时返回#t。因此，撤消标签更改后，移动将立即撤消。（如果需要相反的顺序，请使用begin-edit-sequence和end-edit-sequence创建显式序列。）

系统在不调用此方法的情况下向编辑器添加撤消对象（响应其他方法调用）。

method

(send an-editor adjust-cursor event)

 → (or/c (is-a?/c cursor%) #f)
  event : (is-a?/c mouse-event%)
Specification:

规范：获取要在编辑器显示中使用的光标。如果返回值为#f，则使用默认光标。

另请参见set-cursor。

Default implementation:

默认实现：如果使用set-cursor安装了覆盖光标，则返回已安装的光标。

否则，如果事件是拖动事件，则编辑器中的一个剪切具有焦点，而剪切的调整光标方法返回光标，则返回该光标。

否则，如果光标位于剪切上，而剪切的adjust-cursor方法返回光标，则返回该光标。

否则，如果使用set-cursor安装了光标，则返回已安装的光标。

否则，如果光标位于编辑器中的单击后退区域上，则返回箭头光标。

最后，如果上述情况都不适用，则返回默认光标。对于文本编辑器，默认光标是I。对于粘贴板编辑器，默认光标为箭头。

method

(send an-editor auto-wrap) → boolean?

(send an-editor auto-wrap auto-wrap?) → void?
  auto-wrap? : any/c

启用或禁用根据on-display-size自动调用set-max-width，或获取自动换行的状态。对于文本编辑器，这具有包装编辑器内容以适合显示编辑器的画布的效果（如果多个画布显示编辑器，则为最宽的画布）。对于剪切板编辑器，“自动包装”只是截断剪切板的区域以匹配其画布显示。

当包装模式更改时，立即调用on-display-size方法以更新编辑器的最大宽度。

自动换行最初被禁用。

method

(send an-editor begin-edit-sequence [undoable?    
  interrupt-streak?]) → void?
  undoable? : any/c = #t
  interrupt-streak? : any/c = #t
Specification:

规范：begin-edit-sequence和end-edit-sequence方法用于括起一组编辑器修改，以便结果都同时显示。这些命令可以被任意深度嵌套。使用这些功能可以大大加快显示更改的速度。

当一个编辑器包含其他编辑器时，使用主编辑器上的begin-edit-sequence和end-edit-sequence也会将一些更改括在子编辑器中，但当子编辑器更改时，效果不如为子编辑器调用begin-edit-sequence和end-edit-sequence。

另请参见refresh-delayed?和in-edit-sequence?，有关编辑序列和刷新请求的信息，请参见编辑器和线程。

如果undoable?标志为#f，则序列中所做的更改不能通过undo方法反转。有关撤消的详细信息，请参阅下面的。undoable?标记仅在使用嵌套序列时对最外层的begin-edit-sequence有效。请注意，对于text%对象，插入的字符insert版本会干扰基于序列的撤消分组。

如果interrupt-streak?标志是#f，序列是最外面的，然后序列之前和之后的特殊操作算作连续操作。例如，在将序列附加到剪贴板中之前和之后进行kill。

撤销细节：undoable?像#f是通过不向撤消日志中添加条目来实现的。例如，假设在编辑器中插入了a，插入了b，然后开始了一个不可撤消的编辑序列，a的颜色为红色，然后编辑序列结束。撤消将删除b，保留颜色为红色的a。

另一个例子是，在下面的交互中，undo将删除a而不是b：

> (define t (new text%))
> (begin
   (send t set-max-undo-history 'forever)
   (send t insert "a")
   (send t insert "b")
   (send t begin-edit-sequence #f #f)
   (send t insert "c" 0 0)
   (send t end-edit-sequence)
   (send t get-text))

"cab"

> (begin
   (send t undo)
   (send t get-text))

"cb"

Default implementation:

默认实现：启动序列。

(send an-editor copy-self-to dest) → void?

  dest : (or/c (is-a?/c text%) (is-a?/c pasteboard%))

将an-editor的属性复制到dest。

an-editor中的每个剪切都会被复制并插入到dest中。此外，dest中还安装了an-editor的文件名、最大撤消历史设置、键盘映射、交互式插入符号阈值和加载时覆盖样式设置。最后，复制an-editor的样式列表，并将该副本安装为dest的样式列表。

method

(send an-editor do-edit-operation op    
  [recursive?    
  time]) → void?
  op : 
(or/c 'undo 'redo 'clear 'cut 'copy 'paste
      'kill 'select-all 'insert-text-box
      'insert-pasteboard-box 'insert-image)
  recursive? : any/c = #t
  time : exact-integer? = 0

执行常规编辑命令。op参数必须是有效的编辑命令，其中之一：

  • 'undo——撤消上一个操作

  • 'redo——撤消上一次撤消

  • 'clear——删除当前所选内容

  • 'cut——剪切

  • 'copy——拷贝

  • 'paste——粘贴

  • 'kill——剪切到当前行的结尾，或者如果所选内容和行尾之间只有空格，则剪切新行

  • 'select-all——选择编辑器中的所有内容

  • 'insert-text-box——在此编辑器中将文本编辑器作为项目插入；另请参见on-new-box

  • 'insert-pasteboard-box——在此编辑器中将粘贴板编辑器作为项目插入；另请参见on-new-box。

  • 'insert-image——从用户处获取文件名，并在此编辑器中将图像作为项目插入；另请参见on-new-image-snip。

如果recursive?不是#f，然后将命令传递给此编辑器的任何活动剪切（即拥有插入符号的剪切）。

有关time参数的讨论，请参见剪切和粘贴时间戳。如果time超出平台特定的时间范围，an exn:fail:contract exception is raised。
