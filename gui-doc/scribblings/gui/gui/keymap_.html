class

keymap% : class?

  superclass: object%

editor<%>对象使用keymap%对象以可扩展的方式将键盘和鼠标序列映射到任意函数。也可以在没有编辑器的情况下使用键映射。keymap%对象包含

  • 从函数名到事件处理过程的映射；以及

  • 从键和鼠标序列到函数名的映射。

使用key-event%对象或mouse-event%对象调用键映射中的处理程序过程。它还提供了另一个值，该值取决于使用键映射的上下文（或者更具体地说，是handle-key-event或handle-mouse-event的参数）。对于与editor<%>对象关联的键映射，额外参数通常是接收键盘或鼠标事件的editor<%>对象。

constructor

(new keymap%) → (is-a?/c keymap%)

创建空的键映射。

method

(send a-keymap add-function name func) → void?

  name : string?
  func : (any/c (is-a?/c event%) . -> . any)

命名一个新的函数来处理事件、响应handle-key-event、handle-key-event或call-function。返回值是过程被忽略。

如果已经有一个函数映射到此名称，它将被替换为给定的函数。

调用函数时，它获取传递给handle-key-event、handle-mouse-event或call-function的参数。对于与编辑器关联的键映射，这通常是目标编辑器。

method

(send a-keymap is-function-added? fname) → boolean?

  fname : string?
如果已通过keymap%将fname添加到此键映射，则返回#t，否则返回#f。

此方法不检查链接的键映射，以查看是否已将函数添加到其中之一。

method

(send a-keymap break-sequence) → void?

如果键映射位于键序列的中间，则清除键映射的状态。例如，用户可能点击了escape，然后切换到另一个窗口；如果escape是键盘序列的一部分，则需要清除键映射状态，因为用户不会完成该序列。

可以使用set-break-sequence-callback安装break回调函数。

method

(send a-keymap call-function name    
  in    
  event    
  [try-chain?]) → boolean?
  name : string?
  in : any/c
  event : (is-a?/c event%)
  try-chain? : any/c = #f

直接调用命名事件处理程序。如果找不到函数或找到的处理程序不想处理事件，则返回#f。否则，返回值是事件处理程序的布尔返回值。

如果找到键映射处理程序过程，则将in和event参数传递给它。

如果try-chain?不是#f，链接到此键的键映射将搜索函数名。如果找不到函数且try-chain?是#f；也会引发异常，但异常处理程序无法转义（请参阅延续与事件调度）。

method

(send a-keymap chain-to-keymap next    
  prefix?) → void?
  next : (is-a?/c keymap%)
  prefix? : any/c

将next与a-keymap链接，next键映射将用于处理非a-keymap处理的事件。

如果prefix?是一个真值，那么在两个键映射映射相同的键序列的情况下，next将优先于已经链接到a-keymap的其他键映射。当一个链接的键映射映射一个作为另一个前缀的键时，总是使用较短的键序列，不管prefix?。

使用chain-to-keymap可以将多个键映射从一个键映射链接起来。当键映射与主键映射链接时，未由主键映射处理的事件将传递到链接的键映射，直到某些链接的键映射处理这些事件。键映射可以在任意非循环图中链接在一起。

键映射链接非常有用，因为对于链接的组，正确处理了多个事件序列。如果不链接，事件序列可以在键映射中生成状态，当在其中一个键映射中调用回调时，必须重置该状态。虽然调用break-sequence方法还调用set-break-sequence-callback回调安装的处理程序，但可以使用break-sequence手动清除此状态。

method

(send a-keymap get-double-click-interval)

 → (integer-in 0 1000000)

返回可分离双击单击的最大毫秒数。

默认间隔是以特定于平台的方式确定的，但可以通过'GRacket:doubleClickTime 偏爱全局覆盖；请参见Preferences。

method

(send a-keymap map-function keyname fname) → void?

  keyname : string?
  fname : string?

使用keyname中的字符串编码序列将输入状态序列映射到函数名。keyname的格式是一系列分号分隔的输入状态；每个状态由一系列修饰符标识符和一个键标识符组成。

修改器标识符为：

  • s:——所有平台：Shift

  • c:——所有平台：Control

  • a:——Mac OS：Option

  • m:——当map-command-as-meta-key方法为#t，Windows：Alt；Unix：Meta；Mac OS：Command

  • d:——Mac OS：Command

  • l:——所有平台：Caps Lock

  • g: ——Windows：Control+Alt作为AltGr，参见get-control+meta-is-altgr in key-event%

  • ?: ——所有平台：允许匹配使用Shift、AltGr/Option和/或Caps Lock时产生的字符（如果可用）；请参阅get-other-shift-key-code in key-event%

如果在状态字符串中没有提到某个特定的修饰符，那么它将匹配该修饰符是否被按下的状态。修饰符前面的~使字符串只匹配未按下相应修饰符的状态。如果状态字符串以:开头，则只有在未按字符串中未提及的Shift、Control、Option、Alt、Meta和Command中的修饰符时，字符串才匹配状态。

键标识符可以是键盘上的字符（例如，a、2、?）或者一个特殊的名字。特殊名称如下：

对于一个特殊的关键字，大写并不重要。然而，大写字母对于单字母键名很重要。此外，单字母ASCII 键名被特殊处理：A和s:a都被视为s:A。但是，当c:包含在不带m:的Windows上时，或当d:包含在Mac OS上时，则ASCII字母不使用s:升序，因为Shift键的升序行为被不带Alt的Control（在Windows上）或Command（在Mac OS上）取消。

一个状态可以匹配一个键映射（或键映射链）中映射的多个状态字符串；当一个状态匹配多个状态字符串时，通过根据特定性对字符串排序来选择映射。一个状态字符串，它提到更多的按修饰符，其排名高于其他状态字符串，如果两个字符串提到相同数量的按修饰符，则提到更多未按修饰符的状态字符串排名更高。最后，一个状态字符串包括?:并且仅与Shift、AltGr/Option和/或Caps Lock的相反用法匹配，并且排名低于所有不依赖的匹配项?:，而需要同时使用Shift和AltGr/Option的则排名更低。如果多个匹配字符串具有相同的秩，则可以任意选择匹配项。

示例：

  • "space" ——无论修改器键的状态如何，只要按下空格键就会匹配。

  • "~c:space" ——只要按下空格键而不按下Control键，就会匹配。

  • "a" ——无论何时键入a，都匹配，而不管修改器键的状态如何（Shift除外）。

  • ":a" ——仅当键入a时不按修改键时匹配。

  • "~c:a" ——每次键入a时都匹配，并且既不按Shift键也不按Control键。

  • "c:m:~g:x" ——只要在Windows上的AltGr不构成Control-Alt组合，只要在键入x时使用Control并按下Alt（Windows）或Meta（Unix），就会匹配。

  • ":esc;:c:c" ——匹配Escape键按下（无修改器），然后是Control-C按下（除Control之外无修改器）。

  • "?:d:+" ——用产生+的键按下命令时匹配，即使产生+通常需要按下Shift键。

调用map-function函数以前缀和完整序列的形式映射特定的键序列会引发异常，但异常处理程序无法转义（请参见延续与事件调度）。

在输入状态映射到名称之前，不必将函数名映射到处理程序；调用时按名称调度处理程序。映射到函数名的事件处理程序可以更改，而不会影响从输入状态到函数名的映射。

Changed in version 1.2 of package gui-lib: 添加g:和~g:支持。

method

(send a-keymap set-grab-key-function f) → void?

  f : 
((or/c string? false?)
 (is-a?/c keymap%)
 any/c
 (is-a?/c key-event%)
 . -> . any)

安装一个回调过程，该过程在键映射将输入与函数名匹配或与输入匹配失败后调用。一次只能安装一个键盘抓取功能。当使用抓取回调将键映射链接到键映射时，将为链接键映射中的匹配项调用回调（当链接键映射没有自己的抓取回调时）。

如果抓取回调返回匹配或不匹配回调的真值，则视为已处理该事件。如果回调返回匹配回调的真值，则键映射不会调用匹配的键映射函数。

回调过程f将被调用为：

(f str keymap editor event)

str参数是匹配回调函数的名称，或者#f是不匹配回调函数的名称。keymap参数是匹配的键映射（可能是链接到安装回调的键映射）或安装回调的键映射。editor和event参数与传递给匹配的键映射函数的参数相同。

键抓取回调函数与remove-grab-key-function一起卸载。
