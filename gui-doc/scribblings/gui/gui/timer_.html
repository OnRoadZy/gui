class

timer% : class?

  superclass: object%

timer%对象封装了基于事件的报警。要使用计时器，请使用timer-callback铮实例化它以执行基于警报的操作，或者派生一个新类并重写notify方法以执行基于警报的操作。用start启动计时器，用stop停止计时器。创建计时器时提供初始interval（毫秒）也会启动计时器。

计时器在事件队列中具有相对较高的优先级。因此，如果计时器延迟设置得足够低，则计时器的重复通知可以抢占用户活动（这可能是为了停止计时器）。对于延迟相对较短的计时器，在notify过程中调用yield以允许有保证的事件处理。

有关事件优先级的更多信息，请参阅事件调度和事件空间。

constructor

(new timer% 
    [[notify-callback notify-callback] 
    [interval interval] 
    [just-once? just-once?]]) 
 → (is-a?/c timer%)
  notify-callback : (-> any) = void
  interval : (or/c (integer-in 0 1000000000) #f) = #f
  just-once? : any/c = #f

当计时器过期时，默认的notify方法将调用notify-callback铮。

如果interval为#f（默认值），则计时器不会启动；在这种情况下，必须显式调用start。如果interval是一个数字（毫秒），那么用interval调用start和just-once?。

method

(send a-timer interval) → (integer-in 0 1000000000)

返回每个计时器过期（计时器运行时）之间的毫秒数。

method

(send a-timer notify) → void?

Specification:

规范：当计时器的警报过期时调用（在事件边界上）。

Default implementation:

默认实现：调用创建对象时提供的notify-callback过程。

method

(send a-timer start msec [just-once?]) → void?

  msec : (integer-in 0 1000000000)
  just-once? : any/c = #f

启动（或重新启动）计时器。如果计时器已在运行，则不会更改其警报时间。

计时器的警报在msec毫秒后过期，此时将调用notify（在事件边界上）。如果just-once?为#f，计时器每msec毫秒过期一次，直到计时器显式停止为止。（更准确地说，每次返回notify后，计时器将在msec毫秒后过期。）否则，计时器仅过期一次。

method

(send a-timer stop) → void?

停止计时器。停止的计时器从不调用notify。如果计时器已过期，但尚未调度要notify的调用，则将从事件队列中删除该调用。
