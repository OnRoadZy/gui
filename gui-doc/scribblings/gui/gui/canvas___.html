interface

canvas<%> : interface?

  implements: subwindow<%>

画布是可以在其上绘制图形和文本的子窗口。画布还接收鼠标和键盘事件。

canvas<%>接口由两个类实现：

  • canvas%——用于任意绘图和事件处理的画布；以及

  • editor-canvas%——用于显示editor<%>对象的画布。

要绘制到画布上，请通过get-dc获取其设备上下文。更新画布有两种基本方法：

  • 绘图通常在画布的on-paint回调期间发生。canvas%类支持从默认的on-paint方法调用的paint-callback初始化参数。

    当窗口系统确定画布必须更新时（例如，当画布首次显示或调整其大小时），将自动调用画布（canvas）的on-paint方法作为事件。使用refresh方法从窗口系统显式触发on-paint调用。（可以调用on-paint之前的多个刷新请求合并为一个on-paint调用。）

    在窗口系统调用 on-paint之前，它可能会擦除画布的背景（请参见erase），具体取决于画布的样式（例如，由canvas%的style初始化参数确定）。即使画布的样式禁止显式清除画布，由于窗口移动和调整大小操作，画布也可能被窗口系统删除。对于透明画布，“擦除”意味着画布的父窗口将显示出来。

  • 绘图也可以在窗口系统的on-paint调用之外的任何时间发生，包括来自画布事件空间的处理程序线程（handler thread）以外的线程。从系统在on-paint回调之外绘制是暂时的，因为窗口活动可以擦除画布，但只要不需要窗口刷新，绘制就持久。

    直接调用on-paint方法与在窗口系统的on-paint回调外部绘制相同。对于canvas%，使用refresh-now强制立即更新画布的内容，否则类似于使用refresh排队更新。

绘制到画布的绘图上下文实际上会呈现到屏幕外的缓冲区中。缓冲区将自动异步刷新到屏幕，通过flush方法显式刷新，或者通过flush-display显式刷新——除非画布已禁用刷新。suspend-flush方法挂起画布的刷新，直到匹配的resume-flush调用；可以嵌套suspend-flush和resume-flush的调用，在这种情况下，将挂起刷新，直到最外层的 suspend-flush被resume-flush平衡。来自窗口系统的on-paint调用隐式包装为suspend-flush和resume-flush调用，就像现在通过refresh-now调用画图过程一样。

对于透明画布，线条和文本平滑可以依赖于用作画布背景的窗口。例如，平滑可能会根据目标上下文是白色还是灰色而改变像素的颜色。但是，只有在画布的屏幕外缓冲区中记录相对较少的绘图命令时，才支持对背景敏感的平滑处理。

method

(send a-canvas accept-tab-focus) → boolean?

(send a-canvas accept-tab-focus on?) → void?
  on? : any/c

获取或设置是否为画布启用选项卡焦点（假定画布不是使用canvas%的'no-focus样式创建的）。当启用选项卡焦点时，当用户使用选项卡和箭头键在框架或对话框的控件之间导航时，画布可以接收键盘焦点。默认情况下，禁用选项卡焦点。

当为canvas%对象启用tab focus时，Tab、鼠标、Enter和Escape键盘事件由帧的默认on-traverse-char方法使用。（此外，对话框的默认方法使用转义键事件。）否则，on-traverse-char允许键盘事件传播到画布。

对于editor-canvas%对象，Tab、鼠标、Enter和Escape键盘事件的处理由允许选项卡退出方法确定。

method

(send a-canvas flush) → void?

与flush-display类似，但如果可能，则限制在画布上。

method

(send a-canvas get-canvas-background)

 → (or/c (is-a?/c color%) #f)
返回当前用于在调用on-paint之前“擦除”画布内容的颜色。另请参见set-canvas-background。

如果画布是用'transparent创建的，则结果是#f，否则它始终是color%对象。

method

(send a-canvas get-dc) → (is-a?/c dc<%>)

method

(send a-canvas get-scaled-client-size)

 → 
dimension-integer? dimension-integer?

返回画布的绘图区域尺寸（以未缩放的像素为单位）——也就是说，不进行缩放（请参见屏幕分辨率和文本缩放），这将隐式应用于画布大小和内容。

例如，当Mac OS上的画布驻留在视网膜显示器上时，它的支持比例为2，因此get-scaled-client-size的结果将是get-client-size结果的两倍。如果将同一画布的帧拖动到非视网膜屏幕，则其支持比例可以更改为1，在这种情况下，get-scaled-client-size和get-client-size将产生相同的值。画布的背景比例是否可以更改取决于平台。

get-scaled-client-size报告的大小可能与canvas%实例中OpenGL绘图的视区大小与'gl样式匹配。但是，在Mac OS上，视区将匹配缩放的大小，除非画布是使用gl-config%规范创建的，该规范通过set-hires-mode调整为高分辨率模式。另请参见get-gl-client-size in canvas%。

Added in version 1.13 of package gui-lib.

method

(send a-canvas min-client-height) → dimension-integer?

(send a-canvas min-client-height h) → void?
  h : dimension-integer?

获取或设置画布用于几何图形管理的最小高度，该高度基于客户端大小而不是整个大小。客户端高度通过区域中的min-height in area<%>来获取或更改，根据需要添加或减去边框和滚动条大小。

当最小高度小于画布的图形最小高度时，将忽略最小高度。有关更多信息，请参见几何管理。

method

(send a-canvas min-client-width) → dimension-integer?

(send a-canvas min-client-width w) → void?
  w : dimension-integer?

获取或设置画布用于几何图形管理的最小宽度，该宽度基于画布的客户端大小，而不是其整个大小。客户端宽度是通过min-width in area<%>获得或更改的，根据需要添加或减去边框和滚动条大小。

当最小宽度小于画布的图形最小宽度时，将忽略最小宽度。有关更多信息，请参见几何管理。

method

(send a-canvas on-char ch) → void?

  ch : (is-a?/c key-event%)
Specification:

规范：当画布接收到键盘事件时调用。另见鼠标和键盘事件。

Default implementation:

默认实现：不执行任何操作。

method

(send a-canvas on-event event) → void?

  event : (is-a?/c mouse-event%)
Specification:

规范：当画布接收到鼠标事件时调用。另请参见鼠标和键盘事件，特别注意某些鼠标事件可能会被丢弃。

Default implementation: 默认实现：不执行任何操作。

method

(send a-canvas on-paint) → void?

Specification:

规格：当画布被曝光或调整大小以便画布中的图像可以重新绘制时调用。

当响应系统公开事件调用on-paint并且只有画布的一部分是新公开的时，由on-paint执行的任何绘图操作都将被剪切到新公开的区域；但是，由get-clipping-region报告的剪切区域不会更改。

Default implementation: 默认实现：不执行任何操作。

method

(send a-canvas on-tab-in) → void?

Specification:

规范：当键盘焦点通过键盘导航事件进入画布时调用。对on-focus法也被称为对焦方法，通常用于对焦更改。当键盘焦点由于导航事件而离开画布时，只调用on-focus。

另请参见accept-tab-focus和on-traverse-char in top-level-window<%>。

Default implementation: 默认实现：不执行任何操作。

method

(send a-canvas resume-flush) → void?

有关画布刷新的信息，请参见canvas<%>。

method

(send a-canvas set-canvas-background color) → void?

  color : (is-a?/c color%)

设置用于在调用on-paint之前擦除画布内容的颜色。（此颜色通常在较低级别与画布关联，因此即使画布的完全刷新被其他活动延迟，也会使用它。）

如果画布是用'transparent样式创建的，an exn:fail:contract exception is raised。

method

(send a-canvas set-resize-corner on?) → void?

  on? : any/c

在Mac OS上，当只有一个滚动条可见时，启用或禁用画布右下角的“调整大小”选项卡的空间。此方法对Windows或Unix没有影响，并且当两个滚动条都不可见或不可见时，它也没有影响。默认情况下，“调整角点大小”处于禁用状态，但当使用'resize-corner样式创建画布时，可以启用它。

method

(send a-canvas suspend-flush) → void?

有关画布刷新的信息，请参见canvas<%>。

请注意，暂停画布的刷新可能会阻止在某些平台上刷新同一框架中的其他窗口。
