interface

top-level-window<%> : interface?

  implements: area-container-window<%>

顶级窗口是一个frame%或dialog%对象。

method

(send a-top-level-window can-close?) → boolean?

Refine this method with augment.

在关闭窗口之前调用（例如，由窗口管理器）。如果返回#f，则不会关闭窗口，否则将调用on-close并关闭窗口（即，窗口是隐藏的，就像用#f调用show）。

show 不调用此方法。

method

(send a-top-level-window can-exit?) → boolean?

Specification:

规范：在on-exit前调用，以检查是否允许出口。有关详细信息，请参见on-exit。

Default implementation:

默认实现：调用can-close?并返回结果。

method

(send a-top-level-window center [direction]) → void?

  direction : (or/c 'horizontal 'vertical 'both) = 'both

如果窗口没有父级，则使其在屏幕上居中。如果它有一个父窗口，则该窗口相对于其父窗口的位置居中。

如果direction为'horizontal，则窗口水平居中。如果direction为'vertical，则窗口垂直居中。如果direction是'both，则窗口在两个方向上居中。

method

(send a-top-level-window get-edit-target-object)

 → (or/c (or/c (is-a?/c window<%>) (is-a?/c editor<%>)) #f)

与get-edit-target-window 类似，但是如果一个编辑器画布有焦点，并且它也显示一个编辑器，则返回编辑器而不是画布。此外，如果编辑器的焦点委托给嵌入的编辑器，则返回嵌入的编辑器。

另请参见get-focus-object。

method

(send a-top-level-window get-edit-target-window)

 → (or/c (is-a?/c window<%>) #f)

返回最近具有键盘焦点的窗口，即顶级窗口或其当前显示的子窗口之一。如果窗口及其当前显示的任何子窗口都没有拥有键盘焦点，则返回#f。

另请参见get-focus-window和get-edit-target-object。

method

(send a-top-level-window get-eventspace) → eventspace?

返回窗口的事件空间。

method

(send a-top-level-window get-focus-object)

 → (or/c (or/c (is-a?/c window<%>) (is-a?/c editor<%>)) #f)

与get-focus-window 类似，但如果编辑器画布具有焦点，并且它也显示编辑器，则返回编辑器而不是画布。此外，如果编辑器的焦点委托给嵌入的编辑器，则返回嵌入的编辑器。

另请参见get-edit-target-object。

method

(send a-top-level-window get-focus-window)

 → (or/c (is-a?/c window<%>) #f)

返回具有键盘焦点的窗口，即顶级窗口或其子窗口之一。如果窗口及其任何子窗口都没有焦点，#f将返回。

另请参见get-edit-target-window和get-focus-object。

method

(send a-top-level-window move x y) → void?

  x : position-integer?
  y : position-integer?

将窗口移动到屏幕上的给定位置。

窗户的位置可以被用户拖动窗口更改，这样的更改不会通过这个方法；使用on-move来监视位置更改。

method

(send a-top-level-window on-activate active?) → void?

  active? : any/c

当窗口被激活或停用时调用。当键盘焦点从窗口外部移动到窗口或其子窗口之一时，将激活顶级窗口。当焦点移回窗口外时，它将被禁用。在Mac OS上，浮动框架的子级可以具有焦点，而不是活动非浮动框架的子级；换句话说，浮动框架用作键盘焦点的活动非框架的扩展。

该方法的参数在激活窗口时为#t，在停用窗口时为#f。

method

(send a-top-level-window on-close) → void?

Refine this method with augment.

在窗口关闭前调用（例如，由窗口管理器调用）。show 不调用此方法。

也参见can-close?。

method

(send a-top-level-window on-exit) → void?

Specification:

规范：当操作系统请求关闭应用程序时（例如，在Mac OS的主应用程序菜单中选择退出（Quit）菜单项时），由默认的应用程序退出处理程序调用（由application-quit-handler参数确定）。在这种情况下，对初始事件空间中最近活动的顶级窗口调用此方法，但仅当窗口的can-exit?方法首先返回真。

Default implementation:

默认实现：调用on-close，然后show以隐藏窗口。

method

(send a-top-level-window on-message message) → any/c

  message : any/c
Specification:

规范：一种通用的消息方法，通常通过向窗口发送消息来调用。

如果方法是由send-message-to-window调用的，那么它将在调用send-message-to-window的线程中调用（该线程可能不是窗口的事件空间的处理程序线程）。

Default implementation:

默认实现：返回#<void>。

method

(send a-top-level-window display-changed) → any/c

Specification: 指定：显示配置更改时调用。

要确定新的监视器配置，请使用get-display-count、get-display-size、get-display-left-top-inset和get-display-backing-scale。

请注意，对于监视器的单个逻辑更改，可以多次调用此方法。

Default implementation: 默认实现：返回#<void>。

method

(send a-top-level-window on-traverse-char event) → boolean?

  event : (is-a?/c key-event%)
Specification:

规范：尝试将给定的键盘事件作为导航事件处理，例如移动键盘焦点的Tab键事件。如果处理事件，则返回#t，否则返回#f。 Default implementation:

默认实现：以下规则依次确定event是否处理以及如何处理：

  • 如果当前拥有焦点的窗口专门处理事件，则返回#f。下面介绍窗口类型及其专门处理的键盘事件：

    • editor-canvas%——tab-exit禁用（请参见allow-tab-exit）：当按下Meta（Unix）或Alt（Windows）键时，除字母数字键事件外的所有键盘事件；当tab-exit启用时，除Tab、Enter、Escape和字母数字Meta/Alt事件外的所有键盘事件。

    • canvas%——禁用tab-focus时（请参阅accept-tab-focus）：除按下Meta（Unix）或Alt（Windows）键时发生的字母数字键事件外的所有键盘事件；启用tab-focus时：无键事件

    • text-field%、'single样式——未按下Meta（Unix）或Alt（Windows）键时的箭头键事件和字母数字键事件（以及Mac OS上的所有字母数字事件）

    • text-field%、'multiple样式——除按下Meta（Unix）或Alt（Windows）键时发生的字母数字键事件外的所有键盘事件

    • choice%——未按meta（unix）或alt（windows）键时的箭头键事件和字母数字键事件

    • list-box%——未按meta（unix）或alt（windows）键时的箭头键事件和字母数字键事件

  • 如果event是Tab或箭头键事件，键盘焦点将在窗口中移动，并返回#t。跨平台，通过导航接受键盘焦点的窗口类型可能有所不同，但text-field%窗口始终接受焦点，而message%、gauge%和panel%窗口从不接受焦点。

  • 如果event是一个Space键事件，并且当前拥有焦点的窗口是一个button%、check-box%或radio-box%对象，则该事件的处理方式与单击控件的方式相同，并返回#t。

  • 如果event是Enter键事件，并且当前顶级窗口包含边框按钮，则调用按钮的回调并返回#t。（button%对象的'border样式向用户指示按Enter键与单击按钮相同。）如果窗口不包含边框按钮，如果当前焦点的窗口不是文本字段或编辑器画布，则返回#t。

  • 在对话框中，如果event是一个转义键事件，则该事件的处理方式与单击对话框的关闭框（即对话框的can-close?）和on-close方法被调用，对话框被隐藏），并返回#t。

  • 如果event是字母数字键事件，并且当前顶级窗口包含一个控件，该控件具有与该键匹配的助记键（通过包含&；有关详细信息，请参见get-label）安装，则键盘焦点将移动到匹配的控件。此外，如果匹配的控件是button%、check-box%或radio-box%按钮，则键盘事件的处理方式与单击控件的方式相同。

  • 否则，返回#f。

method

(send a-top-level-window on-system-menu-char event) → boolean?

  event : (is-a?/c key-event%)

检查给定事件是否弹出窗口左上角的系统菜单（仅限Windows）。如果打开窗口的系统菜单，则返回#t，否则返回#f。

method

(send a-top-level-window resize width    
  height) → void?
  width : dimension-integer?
  height : dimension-integer?

设置窗口的大小（以像素为单位），但前提是给定的大小大于窗口的最小大小。

窗口的大小可以被用户更改，这样的更改不会通过这个方法；使用on-size来监视大小更改。

method

(send a-top-level-window set-icon icon    
  [mask    
  which]) → void?
  icon : (is-a?/c bitmap%)
  mask : (is-a?/c bitmap%) = #f
  which : (or/c 'small 'large 'both) = 'both

设置窗口的大图标位图或小图标位图。以后对位图的更改不会影响窗口的图标。

该图标以特定于平台的方式使用：

  • Windows——小图标用于窗口的图标（左上角）和任务栏，大图标用于Alt-Tab任务切换。

  • Mac OS——两个图标都被忽略。

  • Unix——许多窗口管理器使用小图标的方式与Windows相同，而其他窗口管理器在对框架进行图标化时使用小图标；忽略大图标。

两个图标的位图可以是任意大小，但大多数平台将小位图缩放为16 x 16像素，大位图缩放为32 x 32像素。

如果未提供遮罩位图，则整个（矩形）位图将用作图标。

如果提供了遮罩位图，则遮罩必须是单色的。在遮罩位图中，使用黑色像素指示图标的区域，并在图标区域之外使用白色像素。在图标位图中，对图标外的区域使用黑色像素。

method

(send a-top-level-window show show) → void?

  show : any/c

如果该窗口已显示，则它将移动到其他顶级窗口的前面。如果窗口是图标化的（仅限框架），则将其非图标化。

另请参见show in window<%>。
