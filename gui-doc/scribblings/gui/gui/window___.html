interface

window<%> : interface?

  implements: area<%>

window<%>对象是一个具有图形表示的area<%>可以响应事件。

所有window<%>类都接受以下命名的实例化参数：

  • enabled ——默认值为#t；如果是#f，传递到enable。

method

(send a-window accept-drop-files) → boolean?

(send a-window accept-drop-files accept-files?) → void?
  accept-files? : any/c

为窗口启用或禁用拖放，或获取启用状态。最初禁用删除。另请参见on-drop-file。

method

(send a-window client->screen x y)

 → 
position-integer? position-integer?
  x : position-integer?
  y : position-integer?

将本地窗口坐标转换为屏幕坐标。

在Mac OS上，屏幕坐标从菜单栏左上角的(0, 0)开始。相反，move in top-level-window<%>会认为(0, 0)在菜单栏下方。另请参见get-display-left-top-inset。

method

(send a-window enable enable?) → void?

  enable? : any/c

启用或禁用窗口，以便忽略输入事件。（输入事件包括鼠标事件、键盘事件和关闭框单击，但不包括焦点或更新事件。）当窗口被禁用时，对其子级的输入事件也将被忽略。

窗口的启用状态可以被启用父窗口更改，这样的更改不会通过这个方法；使用on-superwindow-enable来监视启用状态更改。

如果enable?为真，则启用该窗口，否则禁用该窗口。

method

(send a-window focus) → void?

如果窗口接受键盘焦点，则将键盘焦点相对于其顶层窗口移动到窗口。如果焦点在窗口的顶层窗口中，或者窗口的顶层窗口可见且浮动（即使用'float样式创建），则焦点将立即移动到此窗口。否则，焦点不会立即移动，但当窗口的顶层窗口获得键盘焦点时，焦点将委托给该窗口。

另请参见on-focus。

注意，在UNIX上，当用户选择菜单项时，键盘焦点可以移动到菜单栏。

当前键盘焦点窗口可以被用户更改，这样的更改不会通过这个方法；使用on-focus来监视焦点更改。

method

(send a-window get-client-handle) → cpointer?

将句柄返回到当前平台的GUI工具箱窗口的“内部”。指针表示的值取决于平台：

  • Windows: HWND

  • Mac OS: NSView

  • Unix: GtkWidget

另请参见get-handle。

method

(send a-window get-client-size)

 → 
dimension-integer? dimension-integer?

获取窗口的内部大小（像素）。对于容器，内部大小是可用于放置子窗口的大小（包括边框边距）。对于画布，这是可见的绘图区域。

客户端大小返回为两个值：宽度和高度（以像素为单位）。

method

(send a-window get-cursor) → (or/c (is-a?/c cursor%) #f)

返回窗口的光标，如果此窗口的光标默认为父级光标，则返回#f。有关详细信息，请参见set-cursor。

method

(send a-window get-handle) → cpointer?

将句柄返回到当前平台的GUI工具箱窗口的“外部”。指针表示的值取决于平台：

  • Windows: HWND

  • Mac OS: NSWindow用于top-level-window<%>对象，NSView用于其他窗口

  • Unix: GtkWidget

另请参见get-client-handle。

method

(send a-window get-height) → dimension-integer?

返回窗口的总高度（像素）。

另请参见reflow-container。

method

(send a-window get-label)

 → 
(or/c label-string?
      (is-a?/c bitmap%)
      (or/c 'app 'caution 'stop)
      (list/c (is-a?/c bitmap%)
              label-string?
              (or/c 'left 'top 'right 'bottom))
      #f)

获取窗口的标签（如果有）。控制窗口通常以某种方式显示其标签。框架和对话框将其标签显示为窗口标题。面板不显示其标签，但标签可用于识别意义。消息、按钮和复选框可以有位图标签（仅当使用位图标签创建时），但所有其他窗口都有字符串标签。此外，消息标签可以是图标符号'app、'caution或'stop，按钮可以同时具有位图标签和字符串标签（以及位图的位置）。

标签字符串可以包含&，它用作Windows和Unix上控件的键盘导航注释。&号不是控件显示标签的一部分；相反，&号将在显示标签中删除（在所有平台上），&号前面的任何字符都加下划线（Windows和Unix），表示该字符是控件的助记键。双&号（&&）转换为单&号（&）（没有显示下划线）。另请参见on-traverse-char。

如果窗口没有标签，则返回#f。

method

(send a-window get-plain-label) → (or/c string? #f)

类似于get-label，除非是：

  • 如果标签包含任何字符c的(&c)，然后删除序列和周围的空白。

  • 如果标签中含有任何字符c的&c，则将&删除。

  • 如果标签包含制表符，则将删除制表符和所有以下字符。

另请参见button%的标签处理。

如果窗口没有标签或窗口标签不是字符串，则返回#f。

method

(send a-window get-size)

 → 
dimension-integer? dimension-integer?

获取整个窗口的当前大小（像素），不计算水平和垂直边距。（在Unix上，此大小不包括框架/对话框的标题栏或边框。）另请参见get-client-size。

几何图形返回为两个值：宽度和高度（以像素为单位）。

另请参见reflow-container。

method

(send a-window get-width) → dimension-integer?

返回窗口当前的总宽度（像素）。

另请参见reflow-container。

method

(send a-window get-x) → position-integer?

返回窗口左边缘在其父坐标系中的位置。

另请参见reflow-container。

method

(send a-window get-y) → position-integer?

返回窗口上边缘在其父坐标系中的位置。

另请参见reflow-container。

method

(send a-window has-focus?) → boolean?

指示窗口当前是否具有键盘焦点。另请参见on-focus。

method

(send a-window is-enabled?) → boolean?

指示窗口当前是否已启用。如果此窗口在启用其祖先时启用，则结果为#t；如果此窗口在启用其祖先时保持禁用，则结果为#f。（也就是说，此方法的结果只受a-window的enable调用的影响，而不受父窗口的启用状态的影响。）

method

(send a-window is-shown?) → boolean?

指示当前是否显示窗口。如果此窗口在显示其祖先时显示，则结果为#t；如果此窗口在显示其祖先时保持隐藏，则结果为#f。（也就是说，此方法的结果只受为a-window的show调用的影响，而不受父窗口的可见性的影响。）

method

(send a-window on-drop-file pathname) → void?

  pathname : path?

当用户将文件拖到窗口上时调用。（在Unix上，通过xnd协议支持拖放。）必须首先为具有accept-drop-files的窗口启用拖放。

在Mac OS上，当应用程序运行并且用户双击应用程序处理的文件或将文件拖到应用程序的图标上时，将调用主线程的应用程序文件处理程序（请参见application-file-handler）。如果为该帧启用了拖放功能，则默认处理程序将调用最近激活的框架的on-drop-file方法，与该框架的事件空间无关（但该方法在该框架的事件空间的处理程序线程中调用）。当应用程序不运行时，文件名作为命令行参数提供。

method

(send a-window on-focus on?) → void?

  on? : any/c
Specification:

规范：当窗口接收或丢失键盘焦点时调用。如果参数为#t，则接收到键盘焦点，否则将丢失。

注意，在UNIX上，当用户选择菜单项时，键盘焦点可以移动到菜单栏。

Default implementation:

默认实现：不执行任何操作。

method

(send a-window on-move x y) → void?

  x : position-integer?
  y : position-integer?
Specification:

规范：在移动窗口时调用。（对于不是顶层窗口的窗口，“移动”表示相对于父窗口的左上角移动。）新位置提供给该方法。

Default implementation:

默认实现：不执行任何操作。

method

(send a-window on-size width height) → void?

  width : dimension-integer?
  height : dimension-integer?
Specification:

规范：在调整窗口大小时调用。该方法提供了窗口的新大小（以像素为单位）。大小值适用于整个窗口，而不仅仅是工作区。

Default implementation:

默认实现：不执行任何操作。

method

(send a-window on-subwindow-char receiver    
  event) → boolean?
  receiver : (is-a?/c window<%>)
  event : (is-a?/c key-event%)
Specification:

规范：当此窗口或子窗口接收到键盘事件时调用。首先调用接收器顶级窗口的on-subwindow-char方法（请参见get-top-level-window）；如果返回值为#f，则为接收器路径中的下一个子项调用on-subwindow-char方法，依此类推。最后，如果服务器的on-subwindow-char方法返回#f，则事件将传递给服务器的常规密钥处理机制。

event参数是为receiver窗口生成的事件。

on-subwindow-event的原子性限制也适用于on-subwindow-char。也就是说，在on-subwindow-char方法上不充分合作可以有效地禁用控件对键盘事件的处理，即使它返回#f。

注意：on-subwindow-char in frame%和on-subwindow-char in dialog%方法使用某些键盘事件（例如，箭头键、Enter）用于在窗口中导航。因为顶层窗口第一次有机会处理键盘事件，所以有些事件永远不会到达“接收器”子级，除非覆盖默认框架或对话框方法。

Default implementation:

默认实现：返回#f。

method

(send a-window on-subwindow-event receiver    
  event) → boolean?
  receiver : (is-a?/c window<%>)
  event : (is-a?/c mouse-event%)
Specification:

规范：当此窗口或子窗口接收到鼠标事件时调用。首先调用接收器顶层on-subwindow-event事件方法（请参见get-top-level-window）；如果返回值为#f，则为接收器路径中的下一个子级调用on-subwindow-event事件方法，依此类推。最后，如果接收器的on-subwindow-event方法返回#f，则事件将传递到接收器的正常鼠标处理机制。

event参数是为receiver窗口生成的事件。

如果on-subwindow-event方法链没有原子地完成（即，不需要运行其他线程）或完成得不够快，则相应的事件可能无法传递到目标控件，例如按钮。换句话说，在on-subwindow-event方法上不充分合作可以有效地禁用控件对鼠标事件的处理，即使控件返回#f。

Default implementation:

默认实现：返回#f。

method

(send a-window on-subwindow-focus receiver    
  on?) → void?
  receiver : (is-a?/c window<%>)
  on? : boolean?
Specification: Called when this window or a child window receives or loses the keyboard focus. This method is called after the on-focus method of receiver. The on-subwindow-focus method of the receiver’s top-level window is called first (see get-top-level-window), then the on-subwindow-focus method is called for the next child in the path to the receiver, and so on. 规范：当此窗口或子窗口接收或丢失键盘焦点时调用。此方法在receiver的on-focus之后调用。首先调用接收器顶层窗口的on-subwindow-focus方法（请参见on-subwindow-focus），然后为接收器路径中的下一个子级调用on-subwindow-focus方法，依此类推。 Default implementation:

默认实现：不执行任何操作。

method

(send a-window on-superwindow-enable enabled?) → void?

  enabled? : any/c
Specification:

规范：每当窗口的启用状态发生更改时，通过事件队列调用，可以通过调用窗口的enable方法，也可以通过启用/禁用窗口的一个祖先来调用。方法的参数指示窗口现在是否已启用。

最初创建窗口时不调用此方法；只有在窗口的初始启用状态更改后才调用此方法。此外，如果启用通知事件排队等待窗口，并在调度事件之前恢复其启用状态，则调度被取消。

如果在删除窗口时窗口祖先的启用状态发生更改（例如，因为它是用delete-child删除的），则不会为已删除窗口排队任何启用事件。但是，如果稍后重新激活窗口，使其处于与取消激活时窗口的状态不同的启用状态，则会立即将启用事件排队。

Default implementation:

默认实现：不执行任何操作。

method

(send a-window on-superwindow-show shown?) → void?

  shown? : any/c
Specification:

规范：每当窗口的可见性发生变化时，通过事件队列调用，可以通过调用窗口的show、显示/隐藏窗口的一个祖先，也可以通过激活或停用窗口或其在容器中的祖先（例如，通过delete-child）。方法的参数指示窗口现在是否可见。

最初创建窗口时不调用此方法；只有在窗口的初始可见性发生更改后才调用此方法。此外，如果显示通知事件排队等待窗口，并且在调度事件之前恢复其可见性，则调度被取消。

Default implementation:

默认实现：不执行任何操作。

method

当弹出菜单时，其目标设置为window。见get-popup-target获取更多信息。

menu在窗口中的位置(x, y)弹出。

method

(send a-window set-cursor cursor) → void?

  cursor : (or/c (is-a?/c cursor%) #f)

设置窗口的光标。提供#f而不是光标值将删除窗口的光标。

如果窗口没有光标，则使用其父窗口的光标。框架和对话框以标准箭头光标开始，文本字段以工字梁光标开始。所有其他窗口都是在没有光标的情况下创建的。

method

(send a-window set-label l) → void?

  l : label-string?

设置窗口标签。更改标签后，窗口的自然最小大小可能不同，但不会重新计算窗口的最小大小。

如果窗口不是用标签创建的，或者窗口是用非字符串标签创建的，则忽略l。

有关详细信息，请参见get-label。

method

(send a-window show show?) → void?

  show? : any/c

显示或隐藏窗口。

窗口的可见性可以被用户单击窗口的关闭框， 例如更改，这样的更改不会通过这个方法；使用on-superwindow-show或on-close来监视可见性更改。

如果show?是#f，窗口是隐藏的。否则，将显示窗口。

method
