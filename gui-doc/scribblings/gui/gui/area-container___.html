interface

area-container<%> : interface?

  implements: area<%>

area-container<%>是一个容器area<%>。

所有area-container<%>类接受以下命名实例化参数:

  • border——默认值为0；传递到border

  • spacing——默认值为0；传递到spacing

  • alignment——默认值是特定于类的，例如vertical-panel%的'(center top)；列表元素将传递到set-alignment

method

(send an-area-container add-child child) → void?

  child : (is-a?/c subwindow<%>)
将给定的子窗口添加到未删除的子窗口集。另请参见change-children。

method

(send an-area-container after-new-child child) → void?

  child : (is-a?/c subarea<%>)
Specification:

此方法在创建新的窗格区域后调用,该区域作为其容器。新子级作为方法的参数提供。

Default implementation:

默认实现:不执行任何操作。

method

(send an-area-container begin-container-sequence) → void?

挂起容器顶级窗口中的几何图形管理,直到调用end-container-sequence。begin-container-sequence和end-container-sequence方法用于将一组容器修改括起来，以便只计算一次生成的几何图形。容器序列还通过change-children延迟显示和隐藏操作，以及通过show在显示的部分屏幕上，直到序列完成。序列开始和结束命令可以任意深地嵌套。

method

(send an-area-container border) → spacing-integer?

(send an-area-container border margin) → void?
  margin : spacing-integer?

获取或设置容器的边框边距(像素)。在计算子区域的位置和大小之前，此边距用作插入面板的工作区。

method

(send an-area-container change-children filter) → void?

  filter : 
((listof (is-a?/c subarea<%>))
 . -> . (listof (is-a?/c subarea<%>)))

采用筛选过程并更改容器的未删除子项列表。筛选过程获取子区域列表并返回新的子区域列表。新列表必须包含创建为此区域的子区域的子区域(即,change-children不能用于更改子区域的父区域)。

更改未删除的子项集后，容器将计算新删除和新未删除的子项集。新删除的窗口将被隐藏。显示新的未删除窗口。

由于不能隐藏非窗口区域，因此不能删除非窗口区域。如果筛选过程删除了非窗口子区域，则会引发异常，并且不会更改未删除的子区域集。

method

(send an-area-container container-flow-modified) → void?

当重写的流定义方法，如place-children的结果更改时，调用此方法。调用通知几何管理器需要重新计算容器子级的位置。

reflow-container方法仅在几何管理器认为自上次计算以来位置已更改时重新计算子位置。

method

(send an-area-container reflow-container) → void?

如果未显示容器窗口，则对容器的子级集所做的更改不一定会立即重新计算容器的大小及其子级的大小和位置。相反，重新计算会延迟到显示容器为止，这样可以避免一系列更改之间的重复计算。reflow-container方法强制立即重新计算容器及其子容器的大小和位置。

在调用reflow-container方法、get-size、get-client-size、get-width、get-height、get-x和get-y之后，立即报告管理器为容器及其子容器应用的大小和位置，即使容器是隐藏的。容器实现可以随时调用诸如get-size之类的函数来获取窗口的当前状态(因为函数不会触发几何管理)。

另请参见container-flow-modified。

method

(send an-area-container set-alignment horiz-align    
  vert-align) → void?
  horiz-align : (symbols 'right 'center 'left)
  vert-align : (symbols 'bottom 'center 'top)
设置容器的对齐规范，该规范确定当容器有剩余空间时(当子容器在特定维度中不可拉伸时)如何定位其子容器。

当容器的水平对齐方式为'left时，子级将在容器中左对齐，空白将插入到右侧。当容器的水平对齐方式为'center时，每个子容器在容器中水平居中。当容器的水平对齐方式为'right时，左上方的空白将插入到左侧。

类似地，容器的垂直对齐方式可以是'top、'center或'bottom。

method

(send an-area-container spacing) → spacing-integer?

(send an-area-container spacing spacing) → void?
  spacing : spacing-integer?

获取或设置容器中子区域之间使用的间距(以像素为单位)。例如，垂直面板在每对垂直对齐的子区域之间插入此间距(顶部或底部没有额外空间)。
