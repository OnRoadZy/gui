9 WXME解码

 (require wxme) package: wxme-lib
The wxme library provides tools for reading WXME editor<%>-format files (see 文件格式) without the racket/gui library.
 (require wxme) package: wxme-lib
wxme库提供不用racket/gui库即可读取WXME editor<%>格式文件（参见文件格式）的工具。

procedure

(is-wxme-stream? in) → boolean?

  in : input-port?

从in探视并返回#t，如果它以表示WXME格式流的魔法字节开始（参见文件格式），否则返回#f。

procedure

(wxme-port->text-port in [close?]) → input-port?

  in : input-port?
  close? : any/c = #t

获取一个输入端口，其流以wxme格式数据开始，并返回一个生成wxme内容文本形式的输入端口，如在drracket中打开wxme文件并将其保存为文本的结果。

与wxme-port->port不同，此函数可能会以使有效程序用无效的方式使用剪切。例如，如果in的wxme流包含位图图像，则可能没有其合理的纯文本版本，因此wxme-port->port可能会将原本有效的Racket程序转换为语法错误的文本，但是，结果可能仍然对人类读卡器或仅在无图形用户界面上下文中运行的近似程序处理工具。

如果close?为真，则关闭结果端口将关闭原始端口。

有关可以读取的非文本内容类型的信息，请参见剪切类映射。

procedure

(wxme-port->port in [close? snip-filter]) → input-port?

  in : input-port?
  close? : any/c = #t
  snip-filter : (any/c . -> . any/c) = (lambda (x) x)

接受其流以WXME格式数据开头的输入端口，并返回一个输入端口，该端口生成转换为字节的文本内容，而非文本内容作为“特殊”值（请参见read-char-or-special）。

新输入端口产生的这些特殊值与将文件读取到editor<%>对象中产生的值不同。特殊值不是snip%的实例，而是object%的简单扩展。有关可以读取的非文本内容类型的信息，请参见剪切类映射。

如果close?为真，那么关闭结果端口会关闭原始端口。

snip-filter过程应用于为流生成的任何特殊值，其结果用作备用特殊值。

如果一个特殊值（可能由过滤过程产生）是一个实现readable<%>接口的对象，则调用该对象的 read-special方法来产生该特殊值。

procedure

(extract-used-classes in) → 
(listof string?) (listof string?)
  in : input-port?

返回两个值：给定流使用的剪切类名列表和流使用的数据类名列表。如果流不是WXME流，则结果是两个空列表。给定流未关闭，并且仅使用WXME流（如果有）的数据。

procedure

(register-lib-mapping! str mod-path) → void?

  str : string?
  mod-path : (cons/c 'lib (listof string?))

将剪切类名映射到提供reader%实现的带引号的模块路径。模块路径的格式必须为'(lib string ...)，其中每个字符串只包含字母数字ASCII字符、.、_、-和空格。

procedure

(string->lib-path str gui?)

 → 
(or/c (cons/c 'lib (listof string?))
      #f)
  str : string?
  gui? : any/c

当gui?是真，为editor<%>模式返回一个str的引用模块路径；当gui?为#f，为wxme模式返回。对于后者，内置映射和映射通过使用register-lib-mapping!注册。如果无法将str解析为库路径，并且没有可用的映射（因为类是内置的或未知的），则结果为#f。

parameter

(unknown-extensions-skip-enabled) → boolean?

(unknown-extensions-skip-enabled skip?) → void?
  skip? : any/c

一个参数。当设置为#f（默认值）时，当WXME流中遇到无法识别的剪切类时，会引发异常。当设置为真值时，无法识别的剪切类的实例只会从转换的流中简单地忽略。

parameter

(broken-wxme-big-endian?) → boolean?

(broken-wxme-big-endian? big?) → void?
  big? : any/c

一个参数。一些旧的和短期的WXME格式依赖于保存文件的机器的字节存储次序。设置此参数以选择读取文件时要使用的字节存储次序；默认值是当前平台的字节存储次序。

procedure

(wxme-read in) → any/c

  in : input-port?

与read类似，但对于以WXME格式数据开头的流而言。如果WXME数据中有多个S表达式，则它们都将被读取并与'begin组合。

如果racket/gui/base可用（由可用的gui-available?确定），然后使用open-input-text-editor。否则，将使用wxme-port->port。

procedure

(wxme-read-syntax source-v in) → (or/c syntax? eof-object?)

  source-v : any/c
  in : input-port?

类似于read-syntax，但对于WXME格式的输入流。如果WXME数据中有多个S表达式，则它们都将被读取并与'begin组合。

如果racket/gui/base可用（由可用的gui-available?确定），然后使用open-input-text-editor。否则，将使用wxme-port->port。

interface

snip-reader<%> : interface?

由读卡器为WXME流中特定类型的数据实现的接口。接口有两种方法：read-header和read-snip。

method

(send a-snip-reader read-header version    
  stream) → any
  version : exact-nonnegative-integer?
  stream : (is-a?/c stream<%>)

每个WXME流最多调用一次以初始化数据类型的流特定信息。这种方法通常不起作用。

method

(send a-snip-reader read-snip text-only?    
  version    
  stream) → 
(if text-only?
    bytes?
    any/c)
  text-only? : boolean?
  version : exact-nonnegative-integer?
  stream : (is-a?/c stream<%>)

在流中遇到数据类型的实例时调用。此方法读取数据并返回将作为解码流的一部分返回的字节，或者返回将作为解码流的“特殊”值返回的任何其他类型的值。结果值可以是实现readable<%>的对象。

调用wxme-port->text-port时text-only?参数为#f，调用wxme-port->port时参数为#t。

interface

readable<%> : interface?

由剪切读取器返回的值实现的接口。唯一的方法是read-special。

method

(send a-readable read-special source    
  line    
  column    
  position) → any/c
  source : any/c
  line : (or/c exact-nonnegative-integer? #f)
  column : (or/c exact-nonnegative-integer? #f)

类似于read-special，但非图形模式。当一个值实现这个接口时，用源位置信息调用它的read-special方法，从WXME解码端口获得“特定”结果。

stream<%> : interface?

表示供snip-reader<%>实例使用的WXME输入流。

读取一个精确的整数，类似于get-exact。

what字段描述在流不继续使用整数的情况下，出于错误消息的目的正在读取的内容。

procedure

(read-snip-from-port name who stream) → bytes?

  name : string?
  who : any/c
  stream : (is-a?/c stream<%>)
给定name（应为剪切类的名称）使用该剪切类在该流中的当前点从给定流中读取。返回处理的字节，与read-snip方法非常相似。

9.1 剪切类映射

当图形数据被封送到WXME格式时，它与剪切类名相关联，以便在加载时与实现相匹配。另请参见剪切类。

理想情况下，剪切类名生成为

(format "~s" (list '(lib string ...)
                   '(lib string ...)))

其中format列表的每个元素都是带引号的模块路径（请参见module-path?）。string只能包含字母数字ASCII字符，加.、_、-和空格，并且不能是"."或".."。

在这种情况下，第一个引用的模块路径用于以图形模式加载WXME文件；相应的模块必须提供实现snip-class%类的snip-class（剪切类）对象。第二个引用的模块路径被wxme库用于转换WXME流，而不需要图形支持；相应的模块必须提供一个实现snip-reader<%>接口的读取器（reader）对象。当然，snip-class%实例和snip-reader<%>实例需要解析相同的格式，但会生成适合不同上下文（即图形或非图形）的不同结果。

如果剪切类名生成为

(format "~s" '(lib string ...))

那么图形模式使用唯一的模块路径，并且wxme需要一个兼容性映射。安装一个带有register-lib-mapping!。

如果剪切类名不具有上述两种格式，则只能在剪切类注册为名称时，或者如果它是内置类之一的名称："wxtext"、"wxtab"、"wximage"或"wxmedia"（用于嵌套编辑器）时，图形模式可以使用数据。如果它不是内置类之一，wxme库需要与register-lib-mapping!一起安装的兼容性映射。

为wxme库自动安装了几个兼容性映射。它们对应于DrRacket的各种版本所支持的流行图形元素，包括注释框、分数、XML框、Racket框、文本框，以及由htdp/image教学包（或者更一般地说，来自mrlib/cache-image-snip）生成的图像，以及测试用例框。

对于由wxme-port->port创建的端口，嵌套的编辑器由wxme/editor库提供的editor%类的实例表示。此类提供了一个单一的方法——get-content-port——它返回编辑器内容的端口。图像表示为wxme/image库提供的image%类的实例。

注释框表示为类的实例，该类扩展了editor%以实现readable<%>；请参见wxme/comment。读取表单生成一个特殊注释（由make-special-comment创建），以便在使用read读取流时，注释框消失；特殊注释内容是可读的实例。XML、Racket和文本框同样生成editor%和readable<%>的实例，这些实例以常规方式扩展；请参见wxme/xml、wxme/scheme和wxme/text。来自htdp/image教学包的图像打包为来自wxme/cache-image库的cache-image%的实例。测试用例框打包为来自wxme/test-case库的test-case%的实例。
