class

editor-snip% : class?

  superclass: snip%

editor-snip%对象是一个snip%对象，它包含并显示一个editor<%>对象。这个剪切类用于将一个编辑器作为单个项插入到另一个编辑器中。

如果editor是非#f，那么它将用作剪切包含的编辑器。另请参见set-editor。

如果with-border?不是#f，则将在剪切周围绘制边界。编辑器显示将按-margin参数中指定的数量插入剪切区域。将使用-inset参数指定的inset绘制边框。

有关嵌入和边距参数的信息，请参见get-inset和get-margin。

Overrides get-extent in snip%.

调用编辑器的get-extent方法，然后添加编辑器剪切的边距。

顶部空间总是对应于编辑器顶行的空间，加上剪切的上边缘。通常，下降对应于编辑最后一行的下降加上剪切的下边距。但是，如果剪切处于对齐顶行模式（请参见set-align-top-line），则下降对应于顶行的下降，加上编辑器行的其余高度，再加上剪切的下边距。

如果编辑器是文本编辑器，那么通常从get-extent返回的编辑器宽度中减去1，因为结果看起来更适合编辑。如果剪切处于文本调整模式（请参见set-tight-text-fit），则从文本编辑器的宽度中减去2，从而消除文本编辑器为闪烁插入符号保留的两个像素。此外，紧致文本适应（tight-text-fit）模式从编辑器高度减去等于行距的量。默认情况下，将禁用紧致文本适应模式。

method

(send an-editor-snip set-editor editor) → void?

  editor : (or/c (or/c (is-a?/c text%) (is-a?/c pasteboard%)) #f)

设置剪切包含的编辑器，释放剪切中的旧编辑器（如果有）。如果新的编辑器已经有了管理员，那么新的编辑器不会安装到剪切中。

当editor-snip%对象未插入到编辑器中时，它没有管理员。在这段时间内，它也不会给所包含的编辑器一个管理员。因此，无管理员包含的编辑器可以“缺陷”到管理员的其他显示。当一个被包含的编辑器出现故障，并且剪切最终被插入到另一个编辑器中时，剪切将删除被包含的故障编辑器，并将其包含的编辑器设置为#f。
