class

image-snip% : class?

  superclass: snip%

image-snip%是可以显示位图图像（通常从文件加载）的剪切。当找不到图像文件时，将绘制一个包含“X”的框。

constructor

(make-object image-snip% [file 
  kind 
  relative-path? 
  inline? 
  backing-scale]) 
 → (is-a?/c image-snip%)
  file : (or/c path-string? input-port? #f) = #f
  kind : 
(or/c 'unknown 'unknown/mask 'unknown/alpha
      'gif 'gif/mask 'gif/alpha
      'jpeg 'png 'png/mask 'png/alpha
      'xbm 'xpm 'bmp 'pict)
 = 'unknown
  relative-path? : any/c = #f
  inline? : any/c = #t
  backing-scale : (>/c 0.0) = 1.0
(make-object image-snip% bitmap [mask]) → (is-a?/c image-snip%)
  bitmap : (is-a?/c bitmap%)
  mask : (or/c (is-a?/c bitmap%) #f) = #f

创建图像剪切，如果指定，则加载图像file（另请参见load-file），或者使用给定的bitmap。

Changed in version 1.1 of package snip-lib: 添加backing-scale参数。

method

(send an-image-snip get-bitmap) → (or/c (is-a?/c bitmap%) #f)

Returns the bitmap that is displayed by the snip, whether set through set-bitmap or load-file. If no bitmap is displayed, the result is #f. 返回由剪切显示的位图，无论是通过set-bitmap还是load-file进行设置。如果没有显示位图，则结果为#f。

method

(send an-image-snip get-filename [relative-path])

 → (or/c path-string? #f)
  relative-path : (or/c (box/c any/c) #f) = #f

返回当前加载的非内联文件的名称，如果文件未加载或文件加载时带有内联（默认值），则返回#f。

The relative-path box is filled with #t如果加载的文件的路径相对于所属编辑器的路径, unless relative-path is #f.

method

(send an-image-snip get-filetype)

 → 
(or/c 'unknown 'unknown/mask 'unknown/alpha
      'gif 'gif/mask 'gif/alpha
      'jpeg 'png 'png/mask 'png/alpha
      'xbm 'xpm 'bmp 'pict)
Returns the kind used to load the currently loaded, non-inlined file, or 'unknown if a file is not loaded or if a file was loaded with inlining (the default). 返回用于加载当前加载的非内联文件的类型，或者如果文件未加载或文件加载时带有内联（默认值），则返回'unknown。

method

(send an-image-snip load-file file    
  [kind    
  relative-path?    
  inline?    
  backing-scale]) → void?
  file : (or/c path-string? input-port? #f)
  kind : 
(or/c 'unknown 'unknown/mask 'unknown/alpha
      'gif 'gif/mask 'gif/alpha
      'jpeg 'png 'png/mask 'png/alpha
      'xbm 'xpm 'bmp 'pict)
 = 'unknown
  relative-path? : any/c = #f
  inline? : any/c = #t
  backing-scale : (>/c 0.0) = 1.0

通过传递file和kind给load-file in bitmap%来加载文件。如果以前使用set-bitmap指定了位图，则该位图（和遮罩）将不再使用。如果file是#f，则当前图像将被清除。

当指定了“unknown/mask”、“gif/mask”或“png/mask”并且加载的位图对象包含一个掩码（请参见获取加载的掩码）时，该掩码用于绘制位图（请参见绘制位图）。但是，建议使用“未知/阿尔法”、“GIF/阿尔法”或“PNG/阿尔法”变体。

如果relative-path?不是#f且file是相对路径，则将使用所属编辑器文件名的路径读取文件。如果图像没有内联，它将保存为相对路径名。

如果inline?不是#f，保存或复制图像时，图像数据将直接保存到文件或剪贴板（保留位图的遮罩，如果有的话）。源文件名和类型不再相关。

Changed in version 1.1 of package snip-lib: 添加backing-scale参数。

method

(send an-image-snip other-equal-to? snip    
  equal?) → boolean?
  snip : (is-a?/c image-snip%)
  equal? : (any/c any/c . -> . boolean?)

如果an-image-snip和snip都有位图且位图相同，则返回#t。如果有一个与主位图尺寸相同的遮罩位图，则遮罩必须相同（或者如果只存在一个遮罩，则它必须对应于实体遮罩）。

给定的equal?函数（用于递归比较）未使用。

method

(send an-image-snip resize w h) → boolean?

  w : (and/c real? (not/c negative?))
  h : (and/c real? (not/c negative?))
Overrides resize in snip%.

位图将被裁剪以适合给定的尺寸。

method

(send an-image-snip set-bitmap bm [mask]) → void?

  bm : (is-a?/c bitmap%)
  mask : (or/c (is-a?/c bitmap%) #f) = #f

设置剪切显示的位图。

绘制位图时使用可选的mask（请参见draw-bitmap），但不推荐直接提供遮罩。如果没有提供遮罩，但位图的get-loaded-mask方法生成的位图尺寸相同，则将其用作遮罩；此外，当将该遮罩保存到文件或复制时（而不保存直接提供的遮罩），该遮罩将与剪切一起保存。但是，通常情况下，bm应该有一个alpha通道，而不是一个单独的遮罩位图。
