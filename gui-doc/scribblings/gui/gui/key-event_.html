class

key-event% : class?

  superclass: event%

key-event%对象包含有关按键或释放事件的信息。关键字事件主要由on-subwindow-char in window<%>和on-char in canvas<%>处理。

对于按键事件，由get-key-code提供虚拟按键代码。对于键释放事件，get-key-code报告'release，get-key-release-code提供虚拟键代码。

另见鼠标和键盘事件。

由get-key-release-code返回的释放键代码初始化为'press。

method

(send a-key-event get-alt-down) → boolean?

Returns #t if the Option (Mac OS) key was down for the event. When the Alt key is pressed in Windows, it is reported as a Meta press (see get-meta-down). 如果事件的Option键（Mac OS）按下，则返回#t。在Windows中按下Alt键时，将报告为Meta按下（请参阅get-meta-down）。

method

(send a-key-event get-caps-down) → boolean?

如果事件的Caps Lock键处于松开状态，则返回#t。

method

(send a-key-event get-control-down) → boolean?

如果事件的Control键已按下，则返回#t。

在Mac OS上，如果按下Control键并单击鼠标按钮，则事件将报告为右键单击同时事件的get-control-down报告#f。

method

(send a-key-event get-control+meta-is-altgr) → boolean?

如果Control+Meta事件应在Windows上被视为AltGr事件，则返回#t。默认情况下，如果Control键是左键，而Alt键(作为Meta)是右键，则AltGr对待请求——用右Alt键在键盘上以这种方式键入，或者由单个AltGr键生成。另请参见any-control+alt-is-altgr，它控制其他Control+Alt组合是否被视为AltGr。

Added in version 1.2 of package gui-lib.

指定的键符号试图去捕捉有用的键，它没有标准的ASCII表示。有几个键有不明显的标准表示。

  • #\space——空格键

  • #\return——Enter或Return键（在所有平台上），但不一定是数字键盘的Enter键（Unix和Mac OS报告为'numpad-enter）

  • #\tab——tab键

  • #\backspace——回退（backspace）键

  • #\rubout——删除（delete）键

如果没有合适的指定键符号或ASCII表示，则会报告#\nul(NUL字符)。

'wheel-up、'wheel-down、'wheel-left或'wheel-right事件可以发送到键盘焦点所在窗口以外的其他窗口，因为某些平台会根据鼠标指针的位置而不是键盘焦点生成滚轮事件。

在Windows上，当在不使用Alt的情况下按下Control键时，ASCII字符的键代码是小写的，大致取消了Shift键的效果。在Mac OS上，当按下Control键或Command键时，键代码求值不受Caps Lock的影响；在Control情况下，如果通过special-control-key禁用Control键的指定操作，通常使用Caps Lock。在Unix上，当在不使用Alt的情况下按下Control键时，使用键代码求值会受Caps Lock影响。

另请参见get-other-shift-key-code。

Changed in version 6.1.0.8 of package gui-lib: 将数字键盘Enter的报告更改为文档化的'numpad-enter，而不是#\u0003。

method

(send a-key-event get-other-shift-key-code)

 → (or/c char? key-code-symbol? #f)

由于键盘映射是不同的，所以有时在键映射中，要让程序知道如果移位键被不同地切换，键盘将为事件生成的结果是有用的。get-other-shift-key-code方法产生另一个映射，如果备用映射不可用，则返回#f，否则返回与get-key-code相同的结果。

get-other-altgr-key-code方法提供了与Windows和Unix上的AltGr键（即Alt与Control组合）或Mac OS上的Option键有关的相同信息。get-other-shift-altgr-key-code方法报告在Shift和AltGr/Option与实际事件不同的情况下的映射。

get-other-shift-key-code、get-other-altgr-key-code和get-other-shift-altgr-key-code结果所有报告的Caps Lock处于关闭状态的键映射，与实际事件的Caps Lock是否处于打开状态无关。get-other-caps-key-code方法报告的映射，在这种情况下，Caps Lock状态被视为get-key-code结果的相反状态。(Caps Lock通常不起作用，或者与Shift的效果相同，因此涉及Caps Lock和其他修改键的进一步组合通常不会产生更多的选择。)

备用映射不适用于所有事件。在Windows上，当生成ASCII字母、ASCII数字和ASCII符号时，会报告备用映射。在Mac OS和Unix上，通常可以使用备用映射。
