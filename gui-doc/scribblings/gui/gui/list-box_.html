class

list-box% : class?

  superclass: object%

  extends: list-control<%>

列表框允许用户从滚动列表中选择一个或多个字符串项。列表框可以是单个选择控件（如果选择了某个项，则删除上一个选择）或多个选择控件（单击某个项可独立于其他选择打开或关闭该项）。

每当用户更改列表框中的选择时，都会调用列表框的回调过程。当创建每个列表框时，将提供一个回调过程作为初始化参数。

一个列表框可以有多个具有可选列标题的列。列表中的项对应于跨越所有列的行。显示列标题时，用户可以更改列宽。此外，列还可以选择支持用户拖动以更改列的显示顺序，而逻辑顺序保持不变。

列表框的行和列是从0索引的。

另请参见choice%。

如果label不是#f，则用作列表框标签。否则，列表框将不显示其标签。

如果&出现在label中，则特别解析为button%。

choices列表指定要在列表框中显示的项目的初始列表。如果列表框有多个列，则choices将确定第一列的内容，其他列将初始化为空字符串。

当用户通过选择、重新选择、取消选择或双击某个项目来更改列表框选择时，将调用callback过程。当用户双击某个项目时，提供给回调的事件类型为'list-box-dclick，否则为'list-box。

columns列表确定列表框中的列数。只有当style包含'column-headers时，才会显示columns中的列标题。如果style还包含'clickable-headers，则单击头将触发对callback的调用，调用的column-control-event%参数的事件类型为'list-box-column。

style规范必须正好包含以下内容之一：

  • 'single ——创建单个选择列表。

  • 'multiple ——创建多个选择列表，单击一次可取消选择其他项目并选择新项目。当单个选择很常见，但允许多个选择时，将此样式用于列表。

  • 'extended ——创建多个选择列表，通过切换单击的项目，单击可扩展或收缩选择。当多个选择是规则而不是例外时，将此样式用于列表。

'multiple和'extended样式决定了对未修改鼠标单击的平台独立解释，但拖动、移位单击、控制单击等具有平台标准解释。无论平台特定的界面是什么，用户总是可以选择不相交的项目集或取消选择项目（并且不选择任何项目）。在某些平台上，用户可以取消选择'single列表框中的（唯一）选定项。

如果style包括'vertical-label，则列表框使用控件上方的标签创建；如果style不包括'vertical-label（可选包括'horizontal-label），然后标签创建在列表框的左侧。 如果style包括'deleted，则列表框将创建为隐藏的，并且它不会影响其父级的几何体；列表框稍后可以通过调用parent的add-child方法激活。

如果style包含'variable-columns，则可以通过append-column和delete-column更改列表框中的列数。

如果selection是整数，则传递给set-selection以设置初始选择。selection必须小于choices的长度。

font参数确定控件内容的字体，label-font确定控件标签的字体。 有关enabled参数的信息，请参见window<%>。 有关horiz-margin和vert-margin的信息参数，请参见subarea<%>。 有关min-width、min-height、stretchable-width、stretchable-height以及stretchable-height参数，请参见area<%>。

如果column-order参数不是#f，它决定逻辑列最初显示的顺序。有关详细信息，请参见set-column-order。如果style包含'column-headers和'reorderable-headers，则用户可以按显示的顺序重新排序列（但显示顺序不会更改列的逻辑顺序）。

method

(send a-list-box append item [data]) → void?

  item : label-string?
  data : any/c = #f
Overrides append in list-control<%>.

向列表框中添加带有关联“数据”对象的新项。data对象不显示在列表框中；它只是为了方便与get-data一起使用而提供的，可能允许程序员避免除了列表框控件之外管理单独的项到数据的映射。

另请参见append in list-control<%>。

method

(send a-list-box get-data n) → any/c

  n : exact-nonnegative-integer?
Returns the data for the item indexed by n, or #f if there is no associated data. 列表框行是从0索引的。 If n is equal to or larger than the number of choices, an exn:fail:contract exception is raised. 返回按n索引的项的数据，如果没有关联的数据则返回#f。列表框行是从0索引的。 如果n等于或大于选择的数目，an exn:fail:contract exception is raised。

See also append and set-data. 另请参见append和set-data。

method

(send a-list-box is-selected? n) → boolean?

  n : exact-nonnegative-integer?

如果选择n索引的项，则返回#t，否则返回#f。列表框行是从0索引的。 如果n等于或大于选择的数目，an exn:fail:contract exception is raised。

列表框的选择可以被用户单击控件更改，这样的更改不会通过这个方法；使用control回调过程（作为初始化参数提供）来监视选择更改。

method

(send a-list-box select n [select?]) → void?

  n : exact-nonnegative-integer?
  select? : any/c = #t

选择或取消选择项目。对于单个选择列表框中的选择，如果当前选择了其他选项，则会自动取消选择。对于多个选择列表框中的选择，与set-selection不同，其他选择被保留。

如果select?为#f，取消选择由n索引的项；否则将选择该项。列表框行是从0索引的。 如果n等于或大于选择的数目，an exn:fail:contract exception is raised。

列表框的选择可以被用户单击控件更改，这样的更改不会通过这个方法；使用control回调过程（作为初始化参数提供）来监视选择更改。

未调用控件的回调过程。

method

(send a-list-box set-column-order column-order) → void?

设置逻辑列的显示顺序。column-order的每个元素必须通过其逻辑位置标识唯一的列，并且所有逻辑列都必须在列表中表示。

另请参见get-column-order。

method

(send a-list-box set-column-width column    
  width    
  min-width    
  max-width) → void?
  column : exact-nonnegative-integer?
  width : dimension-integer?
  min-width : dimension-integer?
  max-width : dimension-integer?

设置由column标识的列的宽度（在逻辑位置，而不是显示位置），该宽度必须在0到1之间，小于列数。

width参数设置当前的显示宽度，而当用户调整列的大小时，min-width和max-width限制列的宽度。width参数必须不小于min-width且不大于max-width。

列的默认宽度是平台特定的，列表框的最后一列可以扩展到控件的末尾，而不受其请求的大小的影响。

另请参见get-column-width。

(send a-list-box set-data n data) → void?

method

(send a-list-box set-first-visible-item n) → void?

  n : exact-nonnegative-integer?

滚动列表框，使按n索引的项位于列表框显示的顶部。列表框行是从0索引的。 如果n等于或大于选择的数目，an exn:fail:contract exception is raised。

列表框的滚动位置可以通过用户单击控件更改，这样的改变不会通过这个方法。当滚动位置更改通过get-first-visible-item表达异议时程序无法检测。
