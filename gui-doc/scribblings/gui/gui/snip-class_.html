class

snip-class% : class?

  superclass: object%

有用的剪切类是通过实例化snip-class%的派生子类来定义的。从snip-class%派生的类用作剪切的“元类”；每个剪切都与snip-class%的实例关联，作为剪切类。有关派生新剪切类的更多信息，请参见实施新的剪切。

constructor

(new snip-class%) → (is-a?/c snip-class%)

创建一个（无用的）剪切类。

method

(send a-snip-class get-classname) → string?

返回类的名称，一个唯一指定此剪切类的字符串。例如，标准文本剪切类名是"wxtext"。以wx开头的名称是保留的。

剪切类名的格式通常应为"((lib ...)\n(lib ...))"，以启用类的按需加载。有关详细信息，请参见剪切类。

method

(send a-snip-class get-version) → exact-integer?

返回此剪切类的版本。当试图加载包含具有相同类名但版本不同的剪切的文件时，会警告用户。

method

(send a-snip-class read f) → (or/c (is-a?/c snip%) #f)

  f : (is-a?/c editor-stream-in%)
Specification:

规范：从给定的流中读取剪切，返回新创建的剪切作为结果，如果有错误，则返回#f。

Default implementation:

默认实现：返回#f。

method

(send a-snip-class read-header f) → boolean?

  f : (is-a?/c editor-stream-in%)
Specification:

规范：调用以读取标题信息，这些信息可能对此类中读取的每个剪切都有用。每个编辑器读取会话只调用一次此方法，并且仅当流包含此类的头信息时才调用此方法。

如果发生读取错误或其他任何错误，返回值为#f。

另请参见write-header。

Default implementation:

默认实现：返回#t。

method

(send a-snip-class reading-version stream) → exact-integer?

  stream : (is-a?/c editor-stream-in%)

返回为当前从给定流中读取的剪切类指定的版本号。

method

(send a-snip-class set-classname name) → void?

  name : string?

设置类的名称。另请参见get-classname。

method

(send a-snip-class set-version v) → void?

  v : exact-integer?

设置此类的版本。请参见get-version。

method

(send a-snip-class write-header stream) → boolean?

  stream : (is-a?/c editor-stream-out%)
Specification:

规范：调用它来编写头信息，这对于为这个类编写的每个剪切都可能有用。每个编辑器编写会话只调用一次此方法，并且仅当编辑器包含此类中的剪切时才调用此方法。

当读取剪切时，只有当write-header向流中写入一些数据时，才会调用read-header。

如果发生写入错误或其他任何情况，返回值为#f。

Default implementation:

默认实现：返回#t。
