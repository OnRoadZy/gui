class

canvas% : class?

  superclass: object%

  extends: canvas<%>

canvas%对象是用于绘制和处理事件的通用窗口。有关绘制到画布上的信息，请参见画布canvas<%>。

constructor

(new canvas% 
    [parent parent] 
    [[style style] 
    [paint-callback paint-callback] 
    [label label] 
    [gl-config gl-config] 
    [enabled enabled] 
    [vert-margin vert-margin] 
    [horiz-margin horiz-margin] 
    [min-width min-width] 
    [min-height min-height] 
    [stretchable-width stretchable-width] 
    [stretchable-height stretchable-height]]) 
 → (is-a?/c canvas%)
  parent : 
(or/c (is-a?/c frame%) (is-a?/c dialog%)
      (is-a?/c panel%) (is-a?/c pane%))
  style : 
(listof (or/c 'border 'control-border 'combo
              'vscroll 'hscroll 'resize-corner
              'gl 'no-autoclear 'transparent
              'no-focus 'deleted))
 = null
  paint-callback : ((is-a?/c canvas%) (is-a?/c dc<%>) . -> . any)
   = void
  label : (or/c label-string? #f) = #f
  gl-config : (or/c (is-a?/c gl-config%) #f) = #f
  enabled : any/c = #t
  vert-margin : spacing-integer? = 0
  horiz-margin : spacing-integer? = 0
  min-width : (or/c dimension-integer? #f) = #f
  min-height : (or/c dimension-integer? #f) = #f
  stretchable-width : any/c = #t
  stretchable-height : any/c = #t

style参数指示以下一个或多个样式：

  • 'border ——给画布一个薄的边框

  • 'control-border ——为画布提供类似于text-field%控件的边框

  • 'combo ——为画布提供一个类似于combo-field%控件的组合按钮；此样式用于'control-border而不是'hscroll或'vscroll

  • 'hscroll ——启用水平滚动（最初可见但不活动）

  • 'vscroll ——启用垂直滚动（最初可见但不活动）

  • 'resize-corner ——在画布右下角只有一个滚动条可见时为调整控件留出空间

  • 'gl ——为OpenGL绘图创建画布，而不是普通的dc<%>绘图；对get-dc的结果调用get-gl-context方法；此样式通常与'no-autoclear组合使用。

  • 'no-autoclear ——防止窗口系统自动删除画布；有关画布刷新的信息，请参阅canvas<%>

  • 'transparent ——窗口系统通过让其父级显示来“擦除”画布；有关窗口刷新和'transparent和屏幕外缓冲交互的信息，请参阅canvas<%>；如果此标志与'no-autoclear组合使用，则结果未定义。

  • 'no-focus ——阻止画布在单击画布或调用focus方法时接受键盘焦点

  • 'deleted ——将画布创建为初始隐藏状态，而不影响parent的几何图形；稍后可以通过调用parent的add-child方法激活画布

'hscroll和'vscroll样式创建了一个带有最初不活动滚动条的画布。滚动条可以用init-manual-scrollbars或init-auto-scrollbars激活，它们可以用show-scrollbars隐藏和重新显示。

paint-callback参数在on-paint方法上默认调用，使用canvas和get-dc返回的DC作为参数。

label参数为get-label命名画布，但不与画布一起显示。

gl-config参数确定此画布的OpenGL上下文的属性，这是通过画布的绘图上下文获得的。另请参见get-dc和get-gl-context in dc<%>。

有关enabled参数的信息，请参见window<%>。 有关horiz-margin和vert-margin的信息参数，请参见subarea<%>。 有关min-width、min-height、stretchable-width、stretchable-height以及stretchable-height参数，请参见area<%>。

method

(send a-canvas get-gl-client-size)

 → 
dimension-integer? dimension-integer?

对于具有'gl样式”的canvas%实例，以OpenGL单位返回画布的绘图区域尺寸。

其结果与在不使用'gl样式的画布中或在Windows和Unix上get-scaled-client-size相同。在Mac OS上，如果创建时提供的 gl-config%规范未指定高分辨率模式，则结果可能与get-client-size相同。

Added in version 1.16 of package gui-lib.

method

(send a-canvas get-scroll-page which)

 → positive-dimension-integer?
  which : (or/c 'horizontal 'vertical)

获取手动滚动条的当前页面步进大小。如果滚动条未激活或是自动的，则结果为0。

racket[which]参数是'horizontal或'vertical，分别指示是获取水平滚动条的页面步进大小还是垂直滚动条的页面步进大小。

另请参见init-manual-scrollbars。

method

(send a-canvas get-view-start)

 → 
dimension-integer? dimension-integer?

根据水平和垂直滚动条的当前值，如果它们初始化为自动滚动条，获取画布可见部分开始的位置（请参见init-auto-scrollbars）。与get-client-size相结合，应用程序只能有效地重新绘制画布的可见部分。这些值以像素为单位。

如果禁用滚动条或将其初始化为手动滚动条（请参见init-manual-scrollbars），则结果为(values 0 0)。

method

(send a-canvas get-virtual-size)

 → (value dimension-integer? dimension-integer?)
获取可滚动画布区域的设备单位大小（与客户端大小相反，客户端大小是画布当前可见的区域）。这与客户端大小相同（由get-client-size返回），除非滚动条初始化为自动（请参见init-auto-scrollbars）。

method

(send a-canvas init-auto-scrollbars horiz-pixels    
  vert-pixels    
  h-value    
  v-value) → void?
  horiz-pixels : (or/c positive-dimension-integer? #f)
  vert-pixels : (or/c positive-dimension-integer? #f)
  h-value : (real-in 0.0 1.0)
  v-value : (real-in 0.0 1.0)

启用并初始化画布的自动滚动条。只能在分别使用'hscroll或'vscroll样式标志创建的画布中激活水平或垂直滚动条。

使用自动滚动条，程序员指定画布的所需虚拟大小，滚动条被自动处理以允许用户滚动虚拟区域。如果不需要滚动条，滚动条不会自动隐藏；请参见show-scrollbars。

鼠标事件（传递给on-event）的坐标不会根据滚动条的位置进行调整；请使用get-view-start方法查找合适的偏移量。

有关手动滚动条的信息，请参见init-manual-scrollbars。水平和垂直滚动条始终都是手动或自动的，但它们是独立启用的。自动滚动条可以重新初始化为手动，反之亦然。

如果horiz-pixels或vert-pixels为#f，则滚动条在相应方向上未启用，并且画布在该方向上的虚拟大小与其客户端大小相同。

h-value和v-value参数将滚动条的初始值指定为滚动条范围的一部分。0.0值将滚动条初始化为左/上（left/top），1.0值将滚动条初始化为右/下（right/bottom）。

可以通过再次调用此函数来调整虚拟大小。

另请参见on-scroll和get-virtual-size。

method

  v-page : positive-dimension-integer?

启用并初始化画布的手动滚动条。只能在分别使用'hscroll或'vscroll样式标志创建的画布中激活水平或垂直滚动条。

对于手动滚动条，程序员负责管理滚动条的所有细节，滚动条状态对画布的虚拟大小没有影响。相反，画布的虚拟大小与其客户端大小相同。

有关自动滚动条的信息，请参见init-auto-scrollbars。水平和垂直滚动条始终都是手动或自动的，但它们是独立启用的。自动滚动条可以重新初始化为手动，反之亦然。

h-length和v-length参数指定滚动步骤中每个滚动条的长度（即每个滚动条的最大值）。如果其中一个为#f，则在相应的方向上禁用滚动条。

h-page和v-page参数设置页面中滚动条步数，即在滚动条控件中按下值指示器上方或下方时移动的量。

h-value和v-value参数指定滚动条的初始值。

如果h-value大于h-value或v-value大于v-length，将引发an exn:fail:contract exception is raised。（页面步骤可能大于滚动条的总大小。）

另请参见on-scroll和get-virtual-size。

(send a-canvas on-scroll event) → void?

仅当手动滚动条发生更改时才调用此方法（请参阅init-manual-scrollbars），而不是自动滚动条；对于自动滚动条，则调用on-paint方法。

method

(send a-canvas refresh-now [paint-proc    
  #:flush? flush?]) → void?
  paint-proc : ((is-a?/c dc<%>) . -> . any)
   = (lambda (dc) (send a-canvas on-paint))
  flush? : any/c = #t

使用画布的绘图上下文调用paint-proc以立即更新画布（与refresh相反，它只将更新请求排队，由窗口系统自行处理）。

在调用paint-proc之前，对画布禁用刷新。此外，画布将被擦除，除非画布具有'no-autoclear样式。在paint-proc返回后，将启用刷新，如果flush?为真，则立即调用flush。

(send a-canvas show-scrollbars show-horiz?    
  show-vert?) → void?
  show-horiz? : any/c
  show-vert? : any/c

显示或隐藏滚动条以show-horiz?和show-horiz?的指示为准。如果show-horiz?为真并且画布不是用'hscroll样式创建的，an exn:fail:contract exception is raised。同样，如果show-vert?为真，并且画布不是用'vscroll样式创建的，an exn:fail:contract exception is raised。

method

(send a-canvas swap-gl-buffers) → void?

调用get-dc返回的该画布的DC的get-gl-context结果的swap-buffers。

swap-buffers in gl-context<%>获取一个可重入锁，因此对swap-gl-buffers或不同线程或OpenGL上下文上的with-gl-context的嵌套调用可能会阻塞或死锁。

method

  [#:fail fail]) → any
  thunk : (-> any)

传递给定的thunk作为get-dc返回的画布DC的get-gl-context结果的当前调用。如果get-gl-context返回#f，则调用fail。

在call-as-current in gl-context<%>方法中当前的调用会获得一个可重入锁，因此使用with-gl-context或交换不同线程或OpenGL上下文上的swap-gl-buffers进行嵌套调用可能会阻塞或死锁。
