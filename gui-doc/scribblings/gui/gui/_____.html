procedure

(append-editor-font-menu-items menu) → void?

  menu : (or/c (is-a?/c menu%) (is-a?/c popup-menu%))
将菜单项附加到menu以实现一组标准的字体-操作操作，例如更改字体字面或样式。每个菜单项的回调都使用顶级窗口中的get-edit-target-object in top-level-window<%>（通过跟踪父项链查找帧，直到到达帧为止）；如果结果是editor<%>对象，则在编辑器上调用change-style in text%或change-style in pasteboard%。

procedure

(append-editor-operation-menu-items 
  menu 
  [text-only? 
  #:popup-position popup-position]) 
 → void?
  menu : (or/c (is-a?/c menu%) (is-a?/c popup-menu%))
  text-only? : any/c = #t
  popup-position : (or/c #f (list/c (is-a?/c text%) exact-nonnegative-integer?))
   = #f
将菜单项附加到menu以实现标准编辑器操作，如剪切和粘贴。每个菜单项的回调使用get-edit-target-object in top-level-window<%>（通过跟踪父级链查找框架，直到到达框架）；如果结果是一个editor<%>对象，则在编辑器中调用do-edit-operation in editor<%>。

如果text-only?是#f，然后将插入非文本剪切（如Insert Image...（插入图像…））的菜单项附加到菜单。

如果popup-position不是#f，则应调用append-editor-operation-menu-items来构建弹出菜单，并且列表的两个元素应为单击弹出菜单鼠标的text%对象和单击发生的位置。在这种情况下，当单击落在一个非string-snip%的截图上时，Copy（复制）和Cut（剪切）菜单被启用，相应的回调将复制和剪切该剪切。

parameter

(current-text-keymap-initializer)

 → ((is-a?/c keymap%) . -> . any/c)
(current-text-keymap-initializer proc) → void?
  proc : ((is-a?/c keymap%) . -> . any/c)

指定键映射初始化过程的参数。调用此过程初始化text-field%对象或由graphical-read-eval-print-loop创建的text%对象的键映射。

初始值设定项获取键映射对象，但不返回任何内容。默认的初始值设定项将给定的键映射链接到一个内部键映射，该键映射为剪切、复制、粘贴、撤消和全选实现标准文本编辑器键盘和鼠标绑定。鼠标右键映射到释放按钮时弹出编辑菜单。在Unix上，还映射了行首（Ctl-A）和行尾（Ctl-E）。

procedure

(open-input-graphical-file filename) → input-port?

  filename : string?

打开filename（在'binary模式下），检查它是否像编辑器格式的“图形”文件。如果文件看起来不是编辑器文件，则返回启用行计数的文件端口。否则，文件将加载到编辑器中，并使用open-input-text-editor创建结果端口。

procedure

(open-input-text-editor 
  text-editor 
  [start-position 
  end-position 
  snip-filter 
  port-name 
  expect-to-read-all? 
  #:lock-while-reading? lock-while-reading?]) 
 → input-port
  text-editor : (is-a?/c text%)
  start-position : exact-nonnegative-integer? = 0
  end-position : (or/c exact-nonnegative-integer? 'end) = 'end
  snip-filter : ((is-a?/c snip%) . -> . any/c) = (lambda (s) s)
  port-name : any/c = text-editor
  expect-to-read-all? : any/c = #f
  lock-while-reading? : any/c = #f

创建从text-editor中提取其内容的输入端口。位置在start-position和end-position之间的编辑器内容是端口的内容。如果end-position为'end，则内容将一直运行到编辑器结束。如果不是string-snip%对象的剪切跨越start-position或end-position，则整个剪切将贡献给端口。如果一个string-snip%实例跨越start-position，则仅贡献start-position之后剪切的一部分；如果一个string-snip%对象跨越end-position，则仅贡献end-position之前的一部分。

text-editor中string-snip%的实例在生成的端口中生成字符序列。所有其他类型的剪切都传递给snip-filter，以获得端口的“特殊”值。如果剪切作为snip-filter的第一个结果返回，并且剪切是readable-snip<%>的实例，则剪切通过read-special方法为端口生成一个特殊值。如果snip-filter返回任何其他类型的剪切，则复制它以获得特殊结果。最后，将来自snip-filter的非剪切第一个结果直接用作特殊结果。

port name参数用于输入端口的名称。希望通读？参数是性能提示；如果要读取整个端口的流，请使用T。

如果text-editor以以下任何方式更改，则不得使用结果端口：插入一个剪切（请参见after-insert）、删除一个剪切（请参见after-delete）、拆分一个剪切（请参见after-split-snip）、合并剪切（请参见after-merge-snips）或剪切更改其计数（这很少见；请参见recounted）。get-revision-number方法可用于检测任何这些更改。

为了防止这种使用，如果lock-while-reading?参数为真值，则open-input-text-editor将lock text-editor，并在返回前调用begin-edit-sequence并解锁lock他，在安全使用上述方法后调用end-edit-sequence。（在某些情况下，如果使用这些方法始终是安全的，则它不会lock编辑器或将其置于编辑序列中。）

procedure

(open-output-text-editor text-editor    
  [start-position    
  special-filter    
  port-name    
  #:eventspace eventspace]) → output-port
  text-editor : (is-a?/c text%)
  start-position : (or/c exact-nonnegative-integer? (one/of 'end))
   = 'end
  special-filter : (any/c . -> . any/c) = (lambda (x) x)
  port-name : any/c = text-editor
  eventspace : (or/c eventspace? #f) = (current-eventspace)

创建将其内容传递到text-editor的输出端口。内容从位置start-position开始写入text-editor，其中'end表示输出应该从文本编辑器的当前结束位置开始。

如果提供了special-filter，它将应用于使用write-special写入端口的任何值，并将结果插入其位置。如果一个特殊值是一个snip%对象，它将被插入到编辑器中。否则，特殊值将display在编辑器中。

如果为生成的输出端口启用了行计数，那么该端口将报告该行、从该行开始的偏移量以及端口写入数据的编辑器中的位置。

如果eventspace不是#f，那么当输出端口用于eventspace处理程序线程以外的线程时，内容将通过eventspace中的低优先级回调传递到文本编辑器。因此，如果eeventspace对应于编辑器显示的eventspace，那么写入输出端口对于任何线程都是安全的。

如果eventspace是#f，请注意端口的线程安全性很差。内容以编辑序列传递到text-editor，但如果编辑器显示在启用的editor-canvas%中，则编辑顺序不够同步。有关详细信息，请参见编辑器和线程。

procedure

(read-editor-global-header in) → boolean?

  in : (is-a?/c editor-stream-in%)

从中读取数据以初始化以从流中读取编辑器。如果读取成功，则返回值为t，否则返回值为f。

通过调用编辑器的read-from-file方法，可以从流中读取一个或多个编辑器。（要读取的编辑器数量必须事先由应用程序知道。）当读取所有编辑器时，调用read-editor-global-footer。对read-editor-global-header和read-editor-global-footer的调用必须包含要从文件中读取的任何调用，并且一次只能使用这些方法读取一个流，或者使用write-editor-global-header和write-editor-global-footer写入。

当从跨越Racket版本的流中读取时，请在此过程之前使用read-editor-version。

procedure

(read-editor-version in    
  in-base    
  parse-format?    
  [raise-errors?]) → boolean?
  in : (is-a?/c editor-stream-in%)
  in-base : (is-a?/c editor-stream-in-base%)
  parse-format? : any/c
  raise-errors? : any/c = #t

从in-base读取版本信息，其中in-base是in的基础。从in-base分析的版本信息记录在in中，以供以后版本敏感分析。如果版本信息读取成功并且版本受支持，则过程结果为真。

如果in为真，则为初始"WXME"格式指示器检查in-base。当"WXME"已被格式调度代码消耗时，使用#f。

如果raise-errors?为真，则读取错误将触发异常，而不是#f结果。

procedure

(text-editor-load-handler [filename]    
  expected-module-name) → any/c
  filename : path = string
  expected-module-name : (or/c symbol? #f)

此过程是用于current-load的加载处理程序。

处理程序识别Racket编辑器格式文件（请参见文件格式），并对其进行解码以便加载。它通常在GRacket启动时安装（请参阅(part ("(lib scribblings/reference/reference.scrbl)" "running-sa"))）。

处理程序通过文件的前12个字符识别编辑器文件：WXME01‹digit›‹digit› ## 。通过创建一个text%对象，用insert-file将文件加载到对象中，然后使用open-input-text-editor将编辑器内容转换为端口，从而打开此类文件进行加载。以这种方式获取端口后，内容的读取方式基本上与默认的Racket加载处理程序相同。不同的是，编辑器可能包含readable-snip<%>的实例，这些实例通过剪切的read-special“读取”；有关详细信息，请参阅open-input-text-editor。

procedure

(write-editor-global-header out) → boolean?

  out : (is-a?/c editor-stream-out%)

将数据写入到out，初始化它以便将编辑器写入流。如果写入成功，返回值为#t，否则返回值为#f。

通过调用编辑器的write-to-file方法，可以将一个或多个编辑器写入流。当所有编辑器都被写入时，调用write-editor-global-footer。调用write-editor-global-header和write-editor-global-footer必须括起对write-to-file的任何调用，并且一次只能使用这些方法写入一个流，或者使用read-editor-global-header和read-editor-global-footer读取。

要支持跨Racket版本的流，请在此过程之前使用write-editor-version。

另见文件格式。

procedure

(write-editor-version out out-base) → boolean?

  out : (is-a?/c editor-stream-out%)
  out-base : (is-a?/c editor-stream-out-base%)

将版本信息写入正在准备中的out-base，以将编辑器信息写入流out。

out参数当前未使用，但out-base应是基于out。将来，out可能会记录有关版本的信息，以供以后版本敏感的输出使用。

如果写入成功，则结果为#t，否则为#f。
