interface

labelled-menu-item<%> : interface?

  implements: menu-item<%>

labelled-menu-item<%>对象是带有字符串标签（即除分隔符之外的任何菜单项）的menu-item<%>。更具体地说，它是menu-item%（普通菜单项）、checkable-menu-item%（可选中菜单项）或menu%（子菜单）的实例。

method

(send a-labelled-menu-item enable enabled?) → void?

  enabled? : any/c

启用或禁用菜单项。如果项目是子菜单（或菜单栏中的菜单），整个菜单将被禁用，但每个子菜单项目 的is-enabled?方法只有当项被特别禁用时（除了子菜单）才会返回#f。

method

(send a-labelled-menu-item get-help-string)

 → (or/c label-string? #f)

返回菜单项的帮助字符串，如果该项没有帮助字符串，则返回#f。

当项目有help时，字符串可用于向用户显示帮助信息。

method

(send a-labelled-menu-item get-label) → label-string?

返回项的标签。

另请参见set-label and get-plain-label。

method

(send a-labelled-menu-item get-plain-label) → label-string?

与get-label类似，只是标签中的&和制表符的剥离方式与set-label相同。

method

(send a-labelled-menu-item is-enabled?) → boolean?

如果菜单项已启用，则返回#t，否则返回[#f]。

另请参见enable。

method

(send a-labelled-menu-item on-demand) → void?

Specification:

规范：通常在用户单击包含该项的菜单栏（在用户看到任何菜单项之前）、在弹出包含该项的弹出菜单之前或在检查包含该项的菜单栏以进行快捷键绑定之前调用。有关详细信息，请参见on-demand in menu-item-container<%>。

on-demand in menu-item-container<%>中的按需方法可以被重写，这样容器就不会调用其项的on-demand方法。

Default implementation:

默认实现：调用创建对象时提供的demand-callback过程。

method

(send a-labelled-menu-item set-help-string help) → void?

  help : (or/c label-string? #f)

设置菜单项的帮助字符串。使用#f删除项目的帮助字符串。

method

(send a-labelled-menu-item set-label label) → void?

  label : label-string?

设置菜单项的标签。如果项目有快捷方式，则快捷方式不受影响。

如果标签包含&且窗口是控件，则会对标签进行特殊分析；在Windows和Unix上，在显示的菜单中，&后面的字符加下划线，表示键盘助记键。按下菜单栏中菜单名称中带下划线字符的Alt键可选择菜单（通过on-menu-char）。当菜单具有焦点时，助记字符用于导航，而不使用Alt。标签中的&&，将替换为文字（非导航）&。在Mac OS上，标签中的&的解析方式与在Unix和Windows中的解析方式相同，但不显示助记下划线。在Mac OS上，在显示标签之前会删除带圆括号的助记字符（连同任何周围的空格），因为带圆括号的助记字符通常用于非罗马语言。最后，出于历史原因，如果标签包含制表符，则该制表符和所有剩余字符将隐藏在显示的菜单中。所有这些规则都与button%和其他窗口中的标签处理一致。

&始终保留在get-label返回的标签中，但从未保留在get-plain-label返回的标签中。
