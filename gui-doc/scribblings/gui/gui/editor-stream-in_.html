class

editor-stream-in% : class?

  superclass: object%

editor-stream-in%对象用于从文件或其他输入流（如剪贴板）读取编辑器信息。

constructor

(make-object editor-stream-in% base)

 → (is-a?/c editor-stream-in%)
  base : (is-a?/c editor-stream-in-base%)

一个in-stream基，可能是editor-stream-in-bytes-base%对象——必须在base中提供。

method

(send an-editor-stream-in get v) → (is-a?/c editor-stream-in%)

  v : (box/c exact-integer?)
(send an-editor-stream-in get v) → (is-a?/c editor-stream-in%)
  v : (box/c real?)

从流中读取数据，并返回自身。从坏流中读取总是给出0。

The v box is filled with 流中的下一个整数或浮点值。.

method

(send an-editor-stream-in get-bytes [len]) → (or/c bytes? #f)

  len : (or/c (box/c exact-nonnegative-integer?) #f) = #f

与get-unterminated-bytes类似，但最后一个读取字节被假定为nul终止符并丢弃。当通过调用put写入数据而不使用显式字节计数时，使用此方法；当使用显式字节计数写入数据时，使用get-unterminated-bytes。

The len box is filled with 字节字符串的长度加上一（表示终止符）。, unless len is #f.

method

(send an-editor-stream-in get-exact) → exact-integer?

返回流中的下一个整数值。

method

(send an-editor-stream-in get-fixed v)

 → (is-a?/c editor-stream-in%)
  v : (box/c exact-integer?)

The v box is filled with 从流中获取的固定大小整数get-fixed-exact.

method

(send an-editor-stream-in get-fixed-exact) → exact-integer?

从流中获取固定大小的整数。有关更多信息，请参见put-fixed。从坏流中读取总是给出0。

method

(send an-editor-stream-in get-inexact) → real?

返回流中的下一个浮点值。

method

(send an-editor-stream-in get-unterminated-bytes [len])

 → (or/c bytes? #f)
  len : (or/c (box/c exact-nonnegative-integer?) #f) = #f

返回流中的下一个字节字符串。这是从流中读取字节的推荐方法；将put与两个参数（沿字节长度传递）一起使用，写出与此方法匹配的字节。

从坏流读取返回#f或#""。

注意，当put没有给定字节长度时，它包含一个额外的nul终止符字节；使用get-bytes读取此类字节字符串。

The len box is filled with 字节字符串的长度。, unless len is #f.

method

(send an-editor-stream-in jump-to pos) → void?

  pos : exact-nonnegative-integer?

跳到流中的给定位置。

method

(send an-editor-stream-in ok?) → boolean?

如果流已准备好读取，则返回#t，否则返回#f。从错误流读取始终返回0或""。

method

(send an-editor-stream-in remove-boundary) → void?

参见set-boundary。

method

(send an-editor-stream-in set-boundary n) → void?

  n : exact-nonnegative-integer?

将文件读取边界设置为超过当前流位置n个字节。如果试图读取超过此边界，则会发出错误信号。通过调用remove-boundary来移除边界。必须通过调用remove-boundary来平衡对set-boundary的每个调用。

边界有助于防止子例程读取过多的数据，从而导致混淆错误。但是，恶意子例程可以自己调用remove-boundary。

method

(send an-editor-stream-in skip n) → void?

  n : exact-nonnegative-integer?

跳过流中接下来的n个字节。

method

(send an-editor-stream-in tell) → exact-nonnegative-integer?

返回当前流位置。
