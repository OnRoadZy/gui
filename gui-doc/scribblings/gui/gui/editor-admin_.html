class

editor-admin% : class?

  superclass: object%

有关管理员角色的信息，请参阅管理员。editor-admin%类从未直接实例化。它甚至没有被大多数程序员通过派生类实例化；每个editor-canvas%和editor-snip%对象创建自己的管理员。但是，派生此类的新实例以在新上下文中显示编辑器可能很有用。此外，从拥有的编辑器调用现有管理员的方法可能很有用。

若要创建新的editor-admin%类，必须重写此处描述的所有方法。它们都由管理员的编辑器调用。

constructor

(new editor-admin%) → (is-a?/c editor-admin%)

创建（无用的）编辑器管理员。

method

(send an-editor-admin get-dc [x y]) → (or/c (is-a?/c dc<%>) #f)

  x : (or/c (box/c real?) #f) = #f
  y : (or/c (box/c real?) #f) = #f
Specification:

规范：返回显示编辑器的绘图上下文，或当前正在绘制编辑器的上下文。当编辑器未嵌入时，返回的上下文始终是显示编辑器的绘图内容。如果未显示编辑器，则返回#f。

图形上下文的原点也会返回，并转换为编辑器的局部坐标。对于嵌入的编辑器，返回的原点仅在绘制编辑器或接收鼠标或键盘事件时可靠。

The x box is filled with 编辑器坐标中的DC的x坐标原点, unless x is #f. The y box is filled with 编辑器坐标中的DC的y坐标原点, unless y is #f.

另请参见editor-location-to-dc-location in editor<%>和dc-location-to-editor-location in editor<%>。 Default implementation:

默认实现：用0.0填充所有框并返回#f。

method

(send an-editor-admin get-max-view x    
  y    
  w    
  h    
  [full?]) → void?
  x : (or/c (box/c real?) #f)
  y : (or/c (box/c real?) #f)
  w : (or/c (box/c (and/c real? (not/c negative?))) #f)
  h : (or/c (box/c (and/c real? (not/c negative?))) #f)
  full? : any/c = #f
Specification:

规格：与get-view相同，除非编辑器在多个标准显示中可见。如果编辑器有多个显示，则计算一个区域，该区域包括所有显示中的可见区域。

请参见get-view。

Default implementation:

默认实现：用0.0填充所有框。

method

(send an-editor-admin get-view x y w h [full?]) → void?

  x : (or/c (box/c real?) #f)
  y : (or/c (box/c real?) #f)
  w : (or/c (box/c (and/c real? (not/c negative?))) #f)
  h : (or/c (box/c (and/c real? (not/c negative?))) #f)
  full? : any/c = #f
Specification:

规范：获取编辑器在其显示中的可见区域（以编辑器坐标为单位），或获取编辑器顶级显示中查看区域的总体大小（对于嵌入的编辑器）。

如果显示为编辑器画布，请参见reflow-container。编辑器画布中的查看区域不是画布的完整客户端区域，因为编辑器画布在客户端区域中显示的编辑器周围安装空白边框。

编辑器可见区域的计算基于顶级显示的当前大小和滚动条值。对于编辑器画布显示，get-view报告的区域不取决于画布是隐藏的、被其他窗口遮挡的还是从屏幕边缘移开的。

The x box is filled with 编辑器坐标中可见区域的左边缘, unless x is #f. The y box is filled with 编辑器坐标中可见区域的上边缘, unless y is #f. The w box is filled with 可见区域的宽度，该宽度可能大于编辑器本身, unless w is #f. The h box is filled with 可见区域的高度，该宽度可能大于编辑器本身, unless h is #f.

如果一个编辑器是完全可见和full?为#f，则x和y都将填充0。

如果full?为真值，则返回的区域是编辑器的顶级显示的视图区域。只有当编辑器嵌入到另一个编辑器中时，此结果才不同；在这种情况下，x和y值可能毫无意义，因为它们位于顶级显示中的即时编辑器坐标系中。

Default implementation:

默认实现：用0.0填充所有框。

method

(send an-editor-admin grab-caret [domain]) → void?

  domain : (or/c 'immediate 'display 'global) = 'global
Specification:

规范：由编辑器调用以请求键盘焦点。如果授予请求，则将调用受管编辑器own-caret方法。

有关domain的可能值的信息，请参阅set-caret-owner。

Default implementation:

默认实现：不执行任何操作。

method

(send an-editor-admin modified modified?) → void?

  modified? : any/c
Specification:

规范：由编辑器调用以报告其修改状态已更改为“已修改”或“未修改”。

另请参见set-modified in editor<%>。

Default implementation:

默认实现：不执行任何操作。

method

(send an-editor-admin needs-update localx    
  localy    
  w    
  h) → void?
  localx : real?
  localy : real?
  w : (and/c real? (not/c negative?))
  h : (and/c real? (not/c negative?))
Specification:

规范：由编辑器调用以请求刷新其显示的表示形式。当管理员决定刷新显示的内容时，它调用refresh方法。

localx、localy、w和h参数指定要更新的编辑器区域（在编辑器坐标中）。

Default implementation:

默认实现：不执行任何操作。

method

(send an-editor-admin popup-menu menu x y) → boolean?

  menu : (is-a?/c popup-menu%)
  x : real?
  y : real?
Specification:

在指定的位置弹出给定的popup-menu%对象坐标（在此窗口的坐标中），然后返回处理未指定数量的事件；菜单可能仍然是此方法返回时弹出。如果从中选择菜单项弹出菜单，调用菜单项的回调。(菜单项回调的事件空间是管理员的显示的事件空间。
当弹出菜单时，其目标设置为此管理员的显示里的顶级编辑器。见get-popup-target获取更多信息。
如果弹出成功，则结果为#t，否则为#f（与用户是否选择弹出菜单中的项目无关）。

菜单显示在编辑器坐标的x和y处。

Default implementation:

默认实现：返回#f。

method

(send an-editor-admin refresh-delayed?) → boolean?

Specification:

规范：如果此管理员显示上的更新当前被延迟，则返回#t（通常通过封闭编辑器中的begin-edit-sequence in editor<%>实现）。

Default implementation:

默认实现：返回#f。

method

(send an-editor-admin resized refresh?) → void?

  refresh? : any/c
Specification:

规范：由编辑器调用，通知其显示编辑器的大小或滚动计数已更改，因此需要调整滚动条以反映新的大小。通常需要在调整大小后更新编辑器，但编辑器决定是否立即进行更新。如果refresh?不是#f，则编辑器请求立即更新。

Default implementation:

默认实现：不执行任何操作。

method

(send an-editor-admin scroll-to localx    
  localy    
  w    
  h    
  [refresh?    
  bias]) → boolean?
  localx : real?
  localy : real?
  w : (and/c real? (not/c negative?))
  h : (and/c real? (not/c negative?))
  refresh? : any/c = #t
  bias : (or/c 'start 'end 'none) = 'none
Specification:

规范：由编辑器调用以请求滚动，以使给定区域可见。编辑器通常需要在滚动后更新，但是编辑器决定是否立即更新。

localx、localy、w和h参数指定要由滚动显示的编辑器区域（在编辑器坐标中）。

如果刷新？不是f，则编辑器请求立即更新。

偏见论点是：

  • 'start ——如果范围不适合可见区域，则显示左上角区域

  • 'none ——无特殊滚动说明

  • 'end ——如果范围不适合可见区域，则显示右下方区域

如果滚动显示，返回值为#t；如果不滚动，返回值为#f（可能是因为请求的区域已经可见，因为显示的大小为零，也可能是因为编辑器当前正在打印）。

如果一个编辑器有多个显示，那么如果任何显示当前具有键盘焦点，则会滚动显示。否则，将滚动编辑器的“主要所有者”（参见call-as-primary-owner）。

Default implementation:

默认实现：返回#f。

method

(send an-editor-admin update-cursor) → void?

Specification:

规范：在此编辑器的显示中为光标排队更新。实际使用的光标将通过调用编辑器的adjust-cursor方法来确定。

Default implementation:

默认实现：不执行任何操作。
