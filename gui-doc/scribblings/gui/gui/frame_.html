class

frame% : class?

  superclass: object%

  extends: top-level-window<%>

框架是顶级容器窗口。它有一个标题栏(显示框架的标签)、一个可选菜单栏和一个可选状态行。

constructor

(new frame% 
    [label label] 
    [[parent parent] 
    [width width] 
    [height height] 
    [x x] 
    [y y] 
    [style style] 
    [enabled enabled] 
    [border border] 
    [spacing spacing] 
    [alignment alignment] 
    [min-width min-width] 
    [min-height min-height] 
    [stretchable-width stretchable-width] 
    [stretchable-height stretchable-height]]) 
 → (is-a?/c frame%)
  label : label-string?
  parent : (or/c (is-a?/c frame%) #f) = #f
  width : (or/c dimension-integer? #f) = #f
  height : (or/c dimension-integer? #f) = #f
  x : (or/c position-integer? #f) = #f
  y : (or/c position-integer? #f) = #f
  style : 
(listof (or/c 'no-resize-border 'no-caption
              'no-system-menu 'hide-menu-bar
              'toolbar-button 'float 'metal
              'fullscreen-button 'fullscreen-aux))
   = null
  enabled : any/c = #t
  border : spacing-integer? = 0
  spacing : spacing-integer? = 0
  alignment : 
(list/c (or/c 'left 'center 'right)
        (or/c 'top 'center 'bottom))
   = '(center top)
  min-width : (or/c dimension-integer? #f) = #f
  min-height : (or/c dimension-integer? #f) = #f
  stretchable-width : any/c = #t
  stretchable-height : any/c = #t

label字符串显示在框架的标题栏中。如果框架的标签已更改（请参见set-label），则标题栏将更新。

parent参数可以是#f或现有框架。在Windows上，如果parent是现有框架，则新框架始终位于其父框架的顶部。在Windows和Unix（对于许多窗口管理器）上，当父级被图标化时，框架会被图标化。

如果parent为#f，则新框架的事件空间是当前事件空间，由current-eventspace确定。否则，parent的事件空间是新框架的事件空间。

如果width或height参数不是#f，则它指定框架的初始大小（以像素为单位），假定大于最小大小，否则使用最小大小。

如果x或y参数不是#f，则指定框架的初始位置。否则，将自动选择一个位置（创建时平铺框架和对话框）。

style标志可调整某些平台上框架的外观：

  • 'no-resize-border——省略窗口周围可调整大小的边框（Windows、Unix）、调整窗口大小的功能（Mac OS）或右下角的增长框（较旧的Mac OS）

  • 'no-caption——省略框架的标题栏（Windows、Mac OS、Unix）

  • 'no-system-menu——省略系统菜单（Windows）

  • 'toolbar-button——包括框架标题栏上的工具栏按钮（Mac OS 10.6及更早版本）；单击工具栏按钮可触发对on-toolbar-button-click调用

  • 'hide-menu-bar——隐藏菜单栏，并在框架处于活动状态（Mac OS）或要求窗口管理器将框架设为全屏（Unix）时停靠。

  • 'float——使框架停留在所有其他非浮动窗口（Windows、Mac OS、Unix）之前；在Mac OS上，浮动框架与活动非浮动框架共享焦点；当此样式与'no-caption组合时，显示框架不会导致键盘焦点移到窗口，在Unix上，单击框架不会移动焦点；在Windows上，浮动框架没有任务栏按钮

  • 'metal——忽略（以前支持Mac OS）

  • 'fullscreen-button——包括框架标题栏上的按钮，将框架置于全屏模式（Mac OS 10.7及更高版本）

  • 'fullscreen-aux——允许框架伴随另一个全屏模式（Mac OS 10.7及更高版本）

即使未显示框架，也可能会在创建框架时排队等待一些通知事件。因此，在处理某些事件或关闭框架的事件空间之前，无法回收新框架的资源（例如内存）。

有关enabled参数的信息，请参见window<%>。 有关border、spacing和alignment的信息参数，请参见area-container<%>。 有关min-width、min-height、stretchable-width、stretchable-height以及stretchable-height参数，请参见area<%>。

Changed in version 6.0.0.6 of package gui-lib: 添加了style的'fullscreen-button和'fullscreen-aux。

method

(send a-frame create-status-line) → void?

在框架底部创建状态行。状态行的宽度是框架的整个宽度（在调整大小时自动调整），高度和文本大小是平台特定的。

另请参见set-status-text。

method

(send a-frame fullscreen fullscreen?) → void?

  fullscreen? : any/c

将框架置于全屏模式或将框架恢复为非全屏模式。框架的显示状态不受影响。

框架的模式可以通过用户更改，这样的改变不会通过这个方法。当框架已经被置于全屏模式通过is-fullscreened?表达异议时程序无法检测。

在Mac OS上，必须使用样式'fullscreen-button创建frame%才能使全屏模式工作，并且需要Mac OS 10.7或更高版本。

Added in version 1.9 of package gui-lib.
Changed in version 1.18: 用#t更改fullscreen，以不暗示在Windows和Mac OS上show

method

(send a-frame get-menu-bar) → (or/c (is-a?/c menu-bar%) #f)

返回框架的菜单栏，或者如果没有为框架创建菜单栏返回#f。

method

(send a-frame has-status-line?) → boolean?

如果框架的状态行已创建，则返回#t，否则返回#f。另请参见create-status-line。

method

(send a-frame iconize iconize?) → void?

  iconize? : any/c

将框架图标化（minimizes）或去图标化（恢复）。去图标化将框架带到前面。

一个框架的图标化可以通过用户更改，这样的改变不会通过这个方法。当一个框架已经图标化通过is-iconized?表达异议时程序无法检测。

method

(send a-frame is-fullscreened?) → boolean?

如果框架处于全屏模式，则返回#t，否则返回#f。

Added in version 6.0.0.6 of package gui-lib.

method

(send a-frame is-iconized?) → boolean?

如果框架被图标化（最小化），则返回#t，否则返回#f。

method

(send a-frame is-maximized?) → boolean?

在Windows和Mac OS上，如果框架最大化，则返回#t，否则返回#f。在Unix上，结果总是#f。

method

(send a-frame maximize maximize?) → void?

  maximize? : any/c
Specification:

规格：在Windows和Mac OS上最大化或还原框架；框架的显示状态不受影响。在Windows上，无法最大化或还原图标化的框架。

窗口的最大化可以被用户更改，这样的更改不会通过这个方法；使用on-size来监视大小更改。

Default implementation:

默认实现：如果maximize?是#f，窗口恢复，否则最大化。

method

(send a-frame modified) → boolean?

(send a-frame modified modified?) → void?
  modified? : any/c

获取或设置框架的修改状态，以反映给用户。在MacOS上，修改状态在框架的关闭按钮中以点的形式反映出来。在Windows和Unix上，修改状态由框架显示标题末尾的星号反映。

method

(send a-frame on-menu-char event) → boolean?

  event : (is-a?/c key-event%)

如果框架具有带键盘快捷键的菜单栏，并且键盘事件包括Control键、Alt键、Option键、Meta键、Command键、Shift键或Function键，则on-menu-char尝试将给定事件与菜单项匹配。如果找到匹配项，则返回#t，否则返回#f。

当匹配对应一个完整的快捷方式组合时，将调用菜单项的回调（在返回on-menu-char之前）。

如果事件不对应于完整的快捷方式组合，则如果事件对应于菜单栏中的助记键，即菜单标题中带下划线的字母，通过在菜单标签中包含一个与符号来安装），则无论如何都会处理该事件。如果找到助记键匹配，键盘焦点将移动到菜单栏（用助记键选择菜单），并返回#t。

method

(send a-frame on-toolbar-button-click) → void?

在Mac OS上，当用户单击用'toolbar-button样式创建的框架上的工具栏按钮时调用。

method

(send a-frame set-status-text text) → void?

  text : string?
Sets the frame’s status line text and redraws the status line. See also create-status-line. 设置框架的状态行文本并重新绘制状态行。另请参见create-status-line。
