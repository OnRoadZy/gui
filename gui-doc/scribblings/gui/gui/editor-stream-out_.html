class

editor-stream-out% : class?

  superclass: object%

editor-stream-out%对象用于将编辑器信息写入文件或其他输出流（如剪贴板）。

constructor

(make-object editor-stream-out% base)

 → (is-a?/c editor-stream-out%)
  base : (is-a?/c editor-stream-out-base%)

输出流基——可能是editor-stream-out-bytes-base%对象——必须在base中被提供。

method

(send an-editor-stream-out jump-to pos) → void?

  pos : exact-nonnegative-integer?
跳到流中的给定位置。

method

(send an-editor-stream-out ok?) → boolean?

如果流已准备好写入，则返回#t，否则返回#f。写入坏流没有效果。

method

(send an-editor-stream-out pretty-finish) → void?

确保流以换行符结尾。此方法由write-editor-global-footer调用。

method

(send an-editor-stream-out pretty-start) → void?

将“注释”写入标识文件格式的流中。此方法由write-editor-global-header调用。

method

(send an-editor-stream-out put n v)

 → (is-a?/c editor-stream-out%)
  n : exact-nonnegative-integer?
  v : bytes?
(send an-editor-stream-out put v)
 → (is-a?/c editor-stream-out%)
  v : bytes?
(send an-editor-stream-out put v)
 → (is-a?/c editor-stream-out%)
  v : exact-integer?
(send an-editor-stream-out put v)
 → (is-a?/c editor-stream-out%)
  v : real?

写入v或v的n字节。

当n与字节字符串v一起提供时，使用get-unterminated-bytes来稍后读取字节。这是写入字节以便以后轻松读取的推荐方法；使用get-unterminated-bytes来重新读取字节。

如果没有提供n且v是一个字节字符串，那么出于历史原因，实际写入的字节数包括一个#\nul终止符，因此使用get-bytes而不是get-unterminated-bytes来稍后读取字节。

method

(send an-editor-stream-out put-fixed v)

 → (is-a?/c editor-stream-out%)
  v : exact-integer?

将固定大小的整数放入流中。这种方法是必要的，因为数字通常是以一种需要不同字节数的方式写入的。在某些情况下，将0临时写入流、写入更多数据，然后返回并将0更改为另一个数字是很有用的；这样的过程需要一个固定大小的数字。

用put-fixed写入流的数字必须用get-fixed-exact或get-fixed读取。

method

(send an-editor-stream-out put-unterminated v)

 → (is-a?/c editor-stream-out%)
  v : bytes?

与用(bytes-length v)和v调用put相同。

method

(send an-editor-stream-out tell) → exact-nonnegative-integer?

返回当前流位置。
