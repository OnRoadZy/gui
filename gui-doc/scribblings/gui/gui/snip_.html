class

snip% : class?

  superclass: object%

  extends: equal<%>

snip%的直接实例是乏味的。有用的剪切是通过实例化派生子类来定义的，但是snip%类定义了基本功能。有关派生新剪切类的更多信息，请参见实施新的剪切。

method

(send a-snip draw dc    
  x    
  y    
  left    
  top    
  right    
  bottom    
  dx    
  dy    
  draw-caret) → void?
  dc : (is-a?/c dc<%>)
  x : real?
  y : real?
  left : real?
  top : real?
  right : real?
  bottom : real?
  dx : real?
  dy : real?
  draw-caret : 
(or/c 'no-caret 'show-inactive-caret 'show-caret
      (cons/c exact-nonnegative-integer?
              exact-nonnegative-integer?))
Specification:

规范：调用（由编辑器）将剪切绘制到给定的绘图上下文中，其中剪切的左上角位于DC坐标中的定位（location）(x, y) 。

参数left、top、right和bottom定义剪切区域（在DC坐标中），剪切可以使用该区域优化绘图，但它也可以忽略这些参数。

dx和dy参数提供可以从x和y中减去的数字，以在编辑器坐标中获得剪切的位置（与用于绘图的DC坐标相反）。

有关draw-caret的信息，请参见插入符号所有权。当draw-caret是对时，不要为选定区域绘制背景，如果(get-highlight-text-color)返回颜色（而不是#f），请使用该颜色绘制选定文本和其他选定前景元素。

在调用此方法之前，将在绘图上下文中设置剪切样式的字体、文本颜色和笔颜色。（绘图上下文没有配置为get-extent或partial-offset。）绘图方法不能对绘图上下文的状态进行任何其他假设，除非剪裁区域已设置为适当的状态。在draw返回之前，它必须恢复它所更改的任何绘图上下文设置。

另请参见on-paint in editor<%>。

当调用此方法时，剪切的编辑器通常在内部锁定以进行写入和回流（另请参见内部编辑器锁定），并且通常处于刷新状态（请参见编辑器和线程）。

Default implementation:

默认实现：不绘制任何内容。

method

(send a-snip get-extent dc    
  x    
  y    
  [w    
  h    
  descent    
  space    
  lspace    
  rspace]) → void?
  dc : (is-a?/c dc<%>)
  x : real?
  y : real?
  w : (or/c (box/c (and/c real? (not/c negative?))) #f) = #f
  h : (or/c (box/c (and/c real? (not/c negative?))) #f) = #f
  descent : (or/c (box/c (and/c real? (not/c negative?))) #f)
   = #f
  space : (or/c (box/c (and/c real? (not/c negative?))) #f) = #f
  lspace : (or/c (box/c (and/c real? (not/c negative?))) #f)
   = #f
  rspace : (or/c (box/c (and/c real? (not/c negative?))) #f)
   = #f
Specification:

规格：计算剪切的宽度、高度、下降（在基线下绘制的高度量）、空位（顶部为“填充”空位的高度量）和水平空位（左右两侧为“填充”空位的宽度量）。通过填充w、h、descent、space、lspace和rspace框返回这些值。

此方法由剪切的管理员调用；通常不由其他人直接调用。要获取剪切的范围，请使用编辑器中的get-snip-location in editor<%>。

为查找字体大小提供了绘图上下文，但不应进行绘图。get-extent和partial-offset方法不得对图形上下文的状态进行任何假设，除非它已正确缩放。特别是，在调用方法之前，剪切样式的字体不会在图形上下文中自动设置。（许多剪切缓存其大小信息，因此自动设置字体将是浪费。）如果get-extent或partial-offset更改了图形上下文的设置，则必须在返回之前还原它们。但是，方法不需要更改绘图上下文；只有字体设置可以影响设备上下文的测量结果，并且在get-text-extent in dc<%>接受一个font%参数来调整大小，该参数将覆盖设备上下文的当前字体。

如果剪切的大小取决于其位置，则剪切的左侧和顶部定位（location）在编辑器坐标中以x和y的形式提供；x和y参数通常被忽略。在文本编辑器中，y坐标是线的顶部定位；在知道剪切的高度之前，可能无法确定剪切的实际顶部定位。

如果剪切缓存结果大小以备将来答复，则在调用size-cache-invalid时（特别是剪切的大小取决于任何设备上下文属性），它应使其缓存的大小无效。

如果剪切的大小在接收到get-extent的调用后发生更改，并且在接收到size-cache-invalid的调用无效之前发生更改，则剪切必须将大小更改通知其管理员，以便管理员可以重新计算其派生的大小信息。通过调用其resized方法通知管理员大小更改。

当调用此方法时，剪切的编辑器通常在内部锁定以进行写入和回流（另请参见内部编辑器锁定）。

Default implementation:

默认实现：用0.0填充所有框。

method

(send a-snip get-flags) → (listof symbol?)

返回定义剪切行为的标志，它是以下符号的列表：

  • 'is-text ——这是从string-snip%派生的文本剪切；不要设置此标志

  • 'can-append ——此剪切可以与同一类型的另一个剪切合并

  • 'invisible ——用户看不到的不可见的剪切，例如换行符

  • 'hard-newline ——换行必须跟随剪切

  • 'newline ——新行当前跟随剪切；只有所属的编辑器才应设置此标志

  • 'handles-events ——此剪切可以处理键盘焦点时的键盘和鼠标事件

  • 'handles-all-mouse-events ——此剪切可以处理触摸剪切或紧跟触摸剪切事件的鼠标事件，即使剪切没有键盘焦点（另请参见on-goodbye-event）。

  • 'handles-between-events ——此剪切处理剪切中项目之间的鼠标事件（而不是默认将鼠标单击视为设置在text%或pasteboard%级别发生的位置或其他事件处理）

  • 'width-depends-on-x ——此剪切的显示宽度取决于编辑器中剪切的x定位；例如：tab

  • 'height-depends-on-y ——此剪切的显示高度取决于编辑器中剪切的y定位

  • 'width-depends-on-y ——此剪切的显示宽度取决于编辑器中剪切的y定位

  • 'height-depends-on-x ——此剪切的显示高度取决于编辑器中剪切的x定位

  • 'uses-editor-path ——此剪切使用其编辑器的路径名，当名称更改时应通知它；通知作为冗余调用提供 给set-admin

(send a-snip get-text offset num [flattened?]) → string?

  offset : exact-nonnegative-integer?
  num : exact-nonnegative-integer?
  flattened? : any/c = #f
Specification:

规范：返回此剪切的文本，从剪切中的位置offset开始，并继续执行num 项的总长度。如果offset大于剪切的计数，则返回""。如果num大于剪切的计数减去偏移量，则返回从偏移量到剪切结尾的文本。

如果flattened?不是#f，则返回扁平文本。关于扁平文本与非扁平文本的讨论，请参见扁平文本。

Default implementation:

默认实现：返回""。

method

(send a-snip merge-with prev) → (or/c (is-a?/c snip%) #f)

  prev : (is-a?/c snip%)
Specification:

规范：将a-snip与prev合并，如果剪切无法合并，则返回#f，否则返回新的合并剪切。只有当两个剪切都来自同一个类并且都具有'can-append标志时，才会调用此方法。

如果返回的剪切没有预期的计数，则强制修改其计数。如果返回的剪切已经由另一个管理员拥有，则会创建一个代理剪切。

调用此方法时，剪切的编辑器通常在内部锁定以供读取（另请参见内部编辑器锁定）。

Default implementation:

默认实现：返回#f。

method

(send a-snip next) → (or/c (is-a?/c snip%) #f)

返回拥有此剪切的编辑器中的下一个剪切，如果这是最后一个剪切，则返回#f。

在文本编辑器中，下一个剪切是该剪切（最后一个）位置之后的位置处的剪切。在粘贴板上，下一个剪切就是紧接着剪切后面的那个。（请参阅编辑器结构和术语以了解有关粘贴板中的剪切顺序的信息。）

  y : real?
  len : exact-nonnegative-integer?
Specification:

规范：从第一个剪切项开始，然后继续为len 项计算剪切的部分宽度。提供了绘图上下文和剪切在编辑器坐标中的定位。另请参见get-extent。

当调用此方法时，剪切的编辑器通常在内部锁定以进行写入和回流（另请参见内部编辑器锁定）。

method

(send a-snip release-from-owner) → boolean?

Specification:

规格：要求剪切尝试从它的所有者那里释放自己。如果剪切未被拥有或发布成功，则返回#t。否则，#f将被退回，而剪切仍归其所有。也参见is-owned?。

使用此方法将剪切从一个编辑器移动到另一个编辑器。此方法通知剪切拥有的编辑器，其他人真的想要控制剪切。从编辑器中删除剪切时，不必使用此方法“清理”剪切。

Default implementation:

默认实现：请求剪切拥有的管理员进行低级发布。

(send a-snip set-admin admin) → void?

  admin : (or/c (is-a?/c snip-admin%) #f)

设置剪切的管理员。只有管理员才能调用此方法。

默认方法将剪切的内部状态设置为记录其管理员。如果剪切已经由管理员拥有，并且管理员没有为转换祝福，则不会修改此状态。如果在text%或pasteboard%的实例对该方法进行敏感调用期间未按预期修改剪切的管理员状态，则可以强制修改内部状态（如果新管理员是#f），或者可以创建代理剪切（如果剪切预期将接收新管理员）。

调用此方法时，剪切（新）编辑器通常在内部锁定以供读取（另请参见内部编辑器锁定）。

method

(send a-snip split position first second) → void?

  position : exact-nonnegative-integer?
  first : (box/c (is-a?/c snip%))
  second : (box/c (is-a?/c snip%))
Specification: 规格：把剪切分成两个剪切。当一个剪切有多个项目，并且在两个项目之间插入了一些东西时，就调用这个函数。

参数是相对位置整数和两个框。位置整数指定应为新的第一个剪切指定多少项；其余项转到新的第二个剪切。这两个箱子必须装上两个新的剪切。（旧的剪切不再使用，因此可以作为新的剪切回收。）

如果返回的剪切没有预期的计数，则强制修改它们的计数。如果返回的剪切已经由另一个管理员拥有，则会创建一个代理剪切。

调用此方法时，剪切的编辑器通常在内部锁定以供读取（另请参见内部编辑器锁定）。

Default implementation:

默认实现：用position元素创建一个新的snip%实例，并修改a-snip以按position减少其计数。将嵌套剪切安装到first，将a-snip安装到second。

将剪切写入给定的流。（剪切读取由剪切类处理）剪切的样式信息（即get-style的内容）将自动保存和恢复。
