mouse-event% : class?

  superclass: event%

mouse-event%对象封装鼠标事件。鼠标事件主要由on-subwindow-event in window<%>和on-event in canvas<%>处理。

另参见鼠标和键盘事件。

  mod5-down : any/c = #f

为特定类型的事件创建鼠标事件。事件类型为：

  • 'enter——鼠标指针进入窗口

  • 'leave——鼠标指针离开窗口

  • 'left-down——按下鼠标左键

  • 'left-up——释放鼠标左键

  • 'middle-down——按下鼠标中键

  • 'middle-up——释放鼠标中键

  • 'right-down——按下鼠标右键（Mac OS：按下控制键单击）

  • 'right-up——释放鼠标右键（Mac OS：按下控制键释放）

  • 'motion——鼠标移动，按下或不按下按钮

有关left-down、middle-down、right-down、x、y、shift-down、control-down、meta-down、alt-down、time-stamp、caps-down、mod3-down、mod4-down和mod5-down的信息，请参见相应的get-和set-方法。

Changed in version 1.1 of package gui-lib: 添加mod3-down、mod4-down和mod5-down。

method

(send a-mouse-event button-changed? [button]) → boolean?

  button : (or/c 'left 'middle 'right 'any) = 'any

如果这是鼠标按钮按下或释放事件，则返回#t，否则返回#f。另见button-up?和button-down?。

如果button不是'any，则仅当它是特定按钮的释放事件时才返回#t。

method

(send a-mouse-event button-down? [button]) → boolean?

  button : (or/c 'left 'middle 'right 'any) = 'any

如果事件是按钮按下，则返回#t，否则返回#f。

如果button不是'any，则仅当它是特定按钮的按下事件时才返回#t。

method

(send a-mouse-event button-up? [button]) → boolean?

  button : (or/c 'left 'middle 'right 'any) = 'any

如果事件用于按钮释放，则返回#t，否则返回#f。（如鼠标和键盘事件中所述，按钮释放事件有时会丢失。）

如果button不是'any，则仅当它是特定按钮的释放事件时才返回#t。

method

(send a-mouse-event dragging?) → boolean?

如果这是拖动事件（按下按钮时的运动），则返回#t，否则返回#f。

method

(send a-mouse-event entering?) → boolean?

如果此事件用于鼠标进入窗口，则返回#t，否则返回#f。

当鼠标按钮启动时，进入/离开事件通知窗口它将开始/停止接收鼠标事件。但是，当鼠标按钮按下时，接收鼠标按下事件的窗口将接收所有鼠标事件，直到释放按钮为止；进入/离开事件不会发送到其他窗口，也不会可靠地传递到点击处理窗口（因为窗口可以通过get-x和get-y检测出其区域外的移动）。另见鼠标和键盘事件。

method

(send a-mouse-event get-alt-down) → boolean?

如果事件的Option键（Mac OS）已按下，则返回#t。在Windows中按下Alt键时，将报告为Meta键按下（请参阅get-meta-down）。

method

(send a-mouse-event get-caps-down) → boolean?

如果事件的Caps Lock键处于打开状态，则返回#t。

method

(send a-mouse-event get-control-down) → boolean?

如果事件的Control键已按下，则返回#t。

在Mac OS上，如果按下控制键并单击鼠标按钮，则事件将报告为右键单击，并且事件报告的get-control-down为#f。
